## Interaction States

### Form States

| State | Trigger | Visual |
|-------|---------|--------|
| Idle | Page load | Form fully rendered, CTA enabled only after Turnstile resolves |
| Validation error | Submit with missing/invalid fields | Red border on field (`border-destructive`), error message below field in destructive color, `aria-invalid="true"` on input, `aria-describedby` pointing to error id |
| Submitting | CTA clicked, Turnstile valid | CTA button shows spinner icon + "Submitting…" label, button disabled, all fields read-only |
| Success | Server returns 200 | Form replaced by success panel (not toast-only — see Success State section) |
| Turnstile blocked | Turnstile not resolved | CTA stays disabled with tooltip "Please complete the security check" |
| Rate limited | Server returns 429 | Toast error: "Too many submissions from this address. Try again in 15 minutes." |
| Server error | Server returns 500 | Toast error + inline message above CTA |
| Already submitted | Duplicate email, Clerk 200 no-op | Success panel (same as fresh submit — no info leak about account existence) |
| Clerk conflict | Existing user | Same success panel — user routed to existing account server-side (INTAKE-06) |

### Field Validation (client-side, fires on blur + on submit)

| Field | Rule | Error copy |
|-------|------|-----------|
| Full name | min 2 chars, max 120 chars | "Please enter your full name." |
| Email | valid email format | "Please enter a valid email address." |
| Role | required selection | "Please select your role." |
| Organization type | required selection | "Please select your organization type." |
| Organization name | min 2 chars, max 200 chars | "Please enter your organization name." |
| Expertise/interest | min 20 chars, max 1000 chars | "Please describe your area of expertise (at least 20 characters)." |
| Turnstile | token present | Not shown as field error — CTA disabled state with tooltip |

### Turnstile Widget Contract

- Mode: **Managed** (not invisible) — government/institutional audience expects to see the security check; invisible mode risks mistrust
- Theme: `light` — matches `#f7fafc` surface
- Size: `normal` (300×65px rendered)
- Placement: Between the last form field ("How did you hear") and the submit button, full-width container
- On success: Turnstile callback stores token in React state; CTA becomes enabled
- On expire: Token cleared, CTA disabled again, widget auto-resets
- On error: Toast "Security check failed. Please refresh and try again." + widget reset

## Error State Copywriting

| Error | Source | User-Visible Copy |
|-------|--------|-------------------|
| Turnstile invalid | Server 403 | "Security verification failed. Please refresh the page and try again." |
| Rate limited | Server 429 | "Too many requests from this email address. Please wait 15 minutes before trying again." |
| Already submitted (same email, no-op) | Server 200 (idempotent) | Show success panel identically — do NOT expose duplicate status |
| Existing Clerk user | Server 200 (INTAKE-06) | Show success panel identically — no info leak |
| Validation: field missing | Client | Field-specific inline copy (see Field Validation table) |
| Network error | fetch failure | "Connection error. Please check your internet and try again." |
| Server error | Server 500 | "Something went wrong on our end. Please try again in a moment. If the problem persists, contact us." |

**Error display pattern:** All server-side errors surface as a `toast.error()` (sonner) AND as an inline `Alert` component above the submit button. This dual presentation ensures screen readers and sighted users both receive the error (WCAG 4.1.3 Status Messages).

## Accessibility Contract (WCAG AA)

| Requirement | Implementation |
|-------------|---------------|
| Focus management | On submit error, focus moves to first invalid field |
| Focus management | On success, focus moves to success heading (tabIndex={-1} + ref.focus()) |
| Color contrast | All text on `#f7fafc` background: foreground `var(--cl-on-surface)` = `#181c1e` — ratio > 7:1 (AAA) |
| Error association | Each field error: `id="fieldname-error"`, input has `aria-describedby="fieldname-error"` and `aria-invalid="true"` |
| Turnstile wrapper | `aria-label="Security check"`, `role="region"` |
| Radio group | `role="radiogroup"`, `aria-labelledby` pointing to "Your Role" section heading |
| Submit disabled state | `aria-disabled="true"` when Turnstile not resolved; tooltip via `title` attribute |
| Loading state | `aria-busy="true"` on form during submission, `aria-live="polite"` on CTA text |
| Success panel | `role="status"`, `aria-live="polite"` — announced to screen readers without focus steal |
| Keyboard nav | All form controls reach via Tab; radio group navigable with arrow keys (RadioGroup default) |
| Touch targets | 44px minimum on all interactive elements on mobile |
| Reduced motion | `prefers-reduced-motion: reduce` respected — Turnstile widget respects this natively |
