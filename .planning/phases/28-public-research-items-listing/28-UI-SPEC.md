## Surface A: `/research/items` Listing Page

**Route:** `app/research/items/page.tsx` (server component, no auth)
**Layout:** Two-column on desktop (240px filter rail + flex-1 card grid). Single column on mobile with collapsible filter panel above grid.

### Layout Structure

```
page wrapper: mx-auto max-w-6xl px-6 py-16
  header: mb-12
    h1: text-[28px] font-semibold leading-[1.2] font-headline
    p: text-sm text-muted-foreground
  body: flex gap-8
    aside (desktop): hidden lg:block w-[240px] shrink-0
      filter panel (rounded-lg border bg-card p-4)
    aside (mobile): lg:hidden collapsible above grid
    main: flex-1 min-w-0
      card grid: grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3
      pagination: mt-8 flex items-center justify-between
```

### Research Card Component

Each card renders `Card` (shadcn) with `h-full transition-colors hover:bg-muted/30` (mirrors `PublicPolicyCard` pattern).

Card anatomy (top to bottom):

1. **Type badge** — `Badge` with `text-xs font-medium` using `--research-status-published-bg` / `--research-status-published-fg` colors (green pill). Text: humanized type label (report, paper, dataset, memo, interview transcript, media coverage, legal reference, case study).
2. **Title** — `text-[20px] font-semibold leading-[1.2]` in `--cl-on-surface` color. Max 2 lines via `line-clamp-2`.
3. **Authors line** — `text-sm text-muted-foreground`. Shows `formatAuthorsForDisplay(item)` output. Anonymous: "Source: Confidential" in `text-muted-foreground`. Named: "Authors: {names}".
4. **Published date** — `text-xs text-muted-foreground`. Format: "Published {MMM d, yyyy}". Rendered as `<time datetime={isoString}>`.
5. **Download/View CTA** — Bottom of card, right-aligned. File-backed: `Button variant="default" size="sm"` with `Download` lucide icon + label. URL-only: `Button variant="outline" size="sm"` with `ExternalLink` lucide icon + label. Each carries descriptive `aria-label` per copywriting contract.

Cards do NOT show: linked sections count, abstract, DOI, internal metadata. Those are detail-page only (CONTEXT.md Q9).

### Filter Panel

Matches Phase 27 `ResearchFilterPanel` pattern (240px desktop rail, collapsible on mobile with `Filter` icon + active-count badge).

Filter controls (all URL-param-driven via server component searchParams):

| Filter | Control | Parameter |
|--------|---------|-----------|
| Document | `Select` (base-ui Select\<string\>) | `?document=` (UUID) |
| Item type | Checkbox group (multi-select, `Checkbox` shadcn) | `?type=` |
| Date from | `Input type="date"` | `?from=YYYY-MM-DD` |
| Date to | `Input type="date"` | `?to=YYYY-MM-DD` |
| Sort | `Select` — newest / oldest | `?sort=newest\|oldest` |

Filter panel heading: `text-sm font-semibold text-foreground mb-3`. Each filter group label: `text-xs font-medium text-muted-foreground uppercase tracking-wide mb-2`. Filter sections separated by `<hr className="border-border my-4">`.

Filter controls are keyboard-navigable. Tab order: document → type checkboxes → from date → to date → sort → apply/clear. Clear filters link visible when any filter is active: `text-sm text-primary underline` with `aria-label="Clear all filters"`.

### Pagination

Simple offset pagination. 40 cards per page (CONTEXT.md SC-1).

```
aria-live="polite" region announcing: "Showing items {start}–{end} of {total}"
Previous button: Button variant="outline" size="sm" disabled when offset=0
Next button: Button variant="outline" size="sm" disabled when (offset + 40) >= total
Page indicator: text-sm text-muted-foreground "Page {n} of {totalPages}"
```

Pagination sits at `mt-8` below the card grid, `flex items-center justify-between`.

### LinkedSectionEntry Component

Each linked section renders as an internal link card:

```

  <div className="flex items-start gap-3 rounded-md border bg-card px-4 py-3 hover:bg-muted/30 transition-colors">
    <FileText className="mt-0.5 size-4 text-muted-foreground shrink-0" />
    <div>
      <p className="text-sm font-medium text-foreground">{sectionTitle}</p>
```

No stakeholder names, no feedback IDs, no internal metadata rendered. Only sectionTitle + documentTitle + relevanceNote.

## Accessibility Requirements

Source: CONTEXT.md SC-7 and ROADMAP.md Phase 28 success criteria.

- Filter controls: all checkboxes have associated `<label>` elements. Date inputs have `aria-label="From date"` / `aria-label="To date"`.
- Pagination: `<nav aria-label="Research items pagination">` wrapper. `aria-current="page"` on current page indicator. `aria-live="polite"` region announces page changes.
- Download CTA: `aria-label="Download {title} ({type})"` on every download button.
- View Source CTA: `aria-label="Open external source for {title} (opens in new tab)"`.
- Card links: linked section entries and version entries have descriptive `aria-label` from their text content.
- Type badges are decorative spans, not interactive — no role needed.
- Back link on detail page: `aria-label="Back to all research items"`.
- Lighthouse target: ≥90 on public detail page (ROADMAP.md SC-7).

## Presigned Download Route

**Route:** `app/api/research/[id]/download/route.ts`

This is a non-UI Route Handler but its response shapes the download button interaction:

- On success: responds with HTTP 302 redirect to the 24h presigned R2 GET URL (not JSON — browser follows directly, triggering file download or navigation).
- On failure (item not published, not file-backed, R2 error): responds with 404 or 500 JSON `{ error: string }`. Client button catches this and shows sonner toast per error state above.
- Rate limit: per-IP via `src/lib/rate-limit.ts`. On rate-limit: 429 response, client shows: "Too many download requests. Please wait a moment and try again."
- The download button uses `window.location.href = /api/research/{id}/download` (not fetch) so the browser handles the redirect natively — no CORS concern.
