       **HocuspocusProvider initialization:**
       - Add `providerRef = useRef<HocuspocusProvider | null>(null)`
       - Add `connectionStatus` state: `'connected' | 'connecting' | 'disconnected'`, default `'connecting'`
       - Check `process.env.NEXT_PUBLIC_HOCUSPOCUS_URL` -- if set, collaboration mode is active
       - In a `useEffect` keyed on `[section.id]`:
         - Get Clerk session token via `session?.getToken()` (async)
         - Create `new HocuspocusProvider({ url: process.env.NEXT_PUBLIC_HOCUSPOCUS_URL!, name: \`section-${section.id}\`, token: token ?? '' })`
         - Subscribe to `onConnect` -> set connectionStatus 'connected'
         - Subscribe to `onDisconnect` -> set connectionStatus 'disconnected'
         - Subscribe to `onClose` -> set connectionStatus 'disconnected'
         - Store in providerRef
         - Cleanup: `provider.destroy()` (CRITICAL: prevents memory leak per research Pitfall 6)
       - If `NEXT_PUBLIC_HOCUSPOCUS_URL` is NOT set, providerRef stays null (single-user fallback)

       **buildExtensions update:**
       - When providerRef.current exists, pass `collaboration` option:
         ```typescript
         collaboration: providerRef.current ? {
           doc: providerRef.current.document,
           provider: providerRef.current,
           user: { name: user?.fullName ?? 'Anonymous', color: getPresenceColor(user?.id ?? '').bg }
         } : undefined
         ```

       **Conditional auto-save (CRITICAL -- per research Pitfall 2):**
       - The `onUpdate: handleUpdate` callback must ONLY fire when collaboration is NOT active
       - When collaboration is active AND connected, auto-save is disabled (Hocuspocus onStoreDocument handles persistence)
       - When collaboration is active BUT disconnected (offline fallback), re-enable auto-save debounce
       - Implementation: wrap `handleUpdate` in a condition: `if (!providerRef.current || connectionStatus === 'disconnected') { debouncedSave(...) }`

       **Content prop behavior:**
       - When collaboration is active, do NOT pass `content` to useEditor (Yjs is the source of truth)
       - When collaboration is NOT active, pass `section.content` as before
       - Implementation: `content: providerRef.current ? undefined : (section.content ?? { type: 'doc', content: [{ type: 'paragraph' }] })`
