## Typography

Inherited from Phase 2. Extended with editor-specific block type sizing.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Default paragraph blocks, list items, table cells |
| Label | 12px | 400 (regular) | 1.4 | Toolbar labels, slash command category headers, placeholder text |
| Heading | 20px | 600 (semibold) | 1.2 | H3 blocks, section titles |
| Display | 28px | 600 (semibold) | 1.2 | H1 blocks in editor |

**Editor block type typography:**

| Block Type | Size | Weight | Line Height | Notes |
|------------|------|--------|-------------|-------|
| H1 | 28px | 600 | 1.2 | Maps to Display role |
| H2 | 24px | 600 | 1.2 | Between Heading and Display |
| H3 | 20px | 600 | 1.2 | Maps to Heading role |
| Paragraph | 14px | 400 | 1.5 | Maps to Body role |
| Blockquote | 14px | 400 | 1.5 | italic, left border 3px muted-foreground |

## Color

Inherited from Phase 2. CSS custom properties from `app/globals.css` (oklch-based neutral palette).

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `oklch(1 0 0)` (white) | `oklch(0.145 0 0)` (near-black) | Editor canvas background, page background |
| Secondary (30%) | `oklch(0.97 0 0)` (neutral-50) | `oklch(0.269 0 0)` (neutral-800) | Toolbar background, slash menu background, callout background, code block background, table header cells |
| Accent (10%) | `oklch(0.205 0 0)` (near-black) | `oklch(0.922 0 0)` (near-white) | See reserved list below |
| Destructive | `oklch(0.577 0.245 27.325)` (red) | `oklch(0.704 0.191 22.216)` (red) | No destructive actions in editor itself |

All color values are referenced via CSS custom properties: `var(--background)`, `var(--primary)`, `var(--muted)`, etc. Never hardcode oklch values directly.

**Accent reserved for these specific elements only:**
1. "Save" / "Save changes" primary CTA button (filled)
2. Active/selected slash command menu item (background highlight)
3. Active toolbar formatting button (bold active, italic active, etc.) — filled state
4. Drag handle hover state (icon color shifts to primary)
5. Focused block outline (1px border-primary on focused block container)
6. Table cell selection highlight (bg-primary/10)

**Do NOT use accent for:** placeholder text (use muted-foreground), callout borders (use border), code block chrome (use muted), toolbar default icon state (use foreground/60).

**Additional semantic colors for editor:**

| Semantic | CSS Variable | Usage |
|----------|-------------|-------|
| Code block bg | `var(--muted)` | Background of code block container |
| Callout bg | `var(--muted)` | Background of callout block |
| Callout border | `var(--border)` | Left border of callout block |
| Blockquote border | `var(--muted-foreground)` | Left border 3px |
| Table border | `var(--border)` | All table cell and outer borders |
| Drag handle default | `var(--muted-foreground)` | GripVertical icon at 40% opacity |
| Placeholder text | `var(--muted-foreground)` | Tiptap placeholder extension |
| Toolbar divider | `var(--border)` | Separator between toolbar groups |

### Component: BlockEditor

**Focal point declaration:** The eye should land first on the editor canvas (white/dominant surface, max-width 768px, generous top padding). The toolbar is secondary — compact at 36px height, anchored above the editor or floating near selection. The slash command menu is transient, appearing only on `/` keypress and dismissed immediately on selection or Escape.

**Structure:**

```
[EditorToolbar — fixed, 36px tall, above editor canvas]
[EditorCanvas — Tiptap EditorContent, max-width 768px, padding xl (32px) top/bottom, padding lg (24px) left/right]
  [Block items with DragHandle on hover (left side)]
  [Block content]
[SlashCommandMenu — popover, appears at cursor position on / key]
[FloatingLinkEditor — popover, appears when link mark is selected]
```

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | All toolbar buttons reachable via Tab. Slash menu navigable with Arrow Up/Down, Enter to select, Escape to close. Editor canvas is the ProseMirror contenteditable element (ARIA role managed by Tiptap). |
| Focus management | On slash menu close (Escape), focus returns to editor cursor position. On link editor close, focus returns to selection. |
| Drag and drop keyboard | Block reorder accessible via keyboard: focus block, use Alt+Arrow Up/Down to move. Announce position: `aria-live="polite"` announces "{Block type} moved to position {N}". |
| Screen reader | Drag handles: `aria-label="Drag to reorder block"`. Toolbar buttons: `aria-label="{action name}"` + `aria-pressed={isActive}` for toggle buttons. Image blocks: require alt text (enforced in image settings). Code blocks: `role="region"` with `aria-label="Code block"`. |
| Color contrast | All text meets WCAG 2.1 AA. oklch neutral palette from globals.css satisfies this. Syntax highlighting colors in code blocks must also meet 3:1 minimum. |
| Motion | All animations respect `prefers-reduced-motion`: disable scale, shadow, and height transitions. Position-only changes remain. Toggle content show/hide becomes instant. |
| Link editor | URL input has `aria-label="URL"`. Apply button has `aria-label="Apply link"`. Remove button has `aria-label="Remove link"`. |
| Image alt text | `alt` attribute is required. ImageBlock settings popover enforces non-empty alt text before allowing image to be inserted. |

## Phase-Specific Notes

### Editor Integration Point

The `SectionContentView` component at `app/(workspace)/policies/[id]/_components/section-content-view.tsx` is replaced by a `BlockEditor` component in edit mode. In read mode, either the existing `SectionContentView` is retained (showing read-only text extraction) or a new read-only Tiptap render is used (EditorContent with `editable: false`). ### Save Strategy

The editor does NOT auto-save on every keystroke (that would require debounced tRPC mutations and potentially too many writes). The save model is:

1. **Explicit save**: "Save changes" button triggers tRPC `updateSectionContent` mutation.
2. **Auto-save on blur**: When editor loses focus (user clicks away), trigger save automatically after 1.5s debounce.
3. **Navigation guard**: If `isDirty` is true (content changed since last save), show native `beforeunload` browser dialog: "You have unsaved changes. Leave anyway?"

The auto-save indicator reflects the last save state (Saving / Saved / Error saving).
