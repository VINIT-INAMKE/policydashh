    **1. Create src/server/services/changeRequest.service.ts** with two exported functions:

    a) `transitionCR(crId: string, event: CREvent, actorId: string)`:
    Replicate the 11-step pattern from feedback.service.ts exactly:
    1. Fetch CR row from changeRequests table
    2. Early exit: if status is 'merged' or 'closed', throw TRPCError BAD_REQUEST with message "CR {readableId} is already {status}"
    3. Restore actor from xstateSnapshot (or fresh start) using the established `as any` pattern
    4. Capture previousState
    5. Send event
    6. Capture newSnapshot / newState
    7. No-op check: if newState === previousState, throw BAD_REQUEST "Invalid transition: cannot apply {event.type} in state {previousState}"
    8. Build updateData: { status: newState, xstateSnapshot, updatedAt }
       - For APPROVE: add approverId from event, approvedAt: new Date()
       - For CLOSE: add closureRationale from event
       - For MERGE: this path should NOT be used (merge goes through mergeCR instead), but if it is, add mergedVersionId
    9. db.update(changeRequests)
    10. db.insert(workflowTransitions) with entityType 'change_request'
    11. actor.stop()
    12. Return updated row

    b) `mergeCR(crId: string, mergeSummary: string, actorId: string)`:
    Single atomic transaction via db.transaction():
    1. Fetch CR -- verify status === 'approved'
    2. Generate next version label: query MAX versionLabel from documentVersions WHERE documentId = cr.documentId, parse "v0.N" -> increment to "v0.{N+1}". If no versions exist, use "v0.1"
    3. Insert documentVersions row: { documentId, versionLabel, mergeSummary, createdBy: actorId, crId }
    4. Update CR: { status: 'merged', mergedVersionId: version.id, mergedBy: actorId, mergedAt: new Date(), xstateSnapshot: null (finalized), updatedAt }
    5. Bulk-update linked feedback: SELECT feedbackId FROM crFeedbackLinks WHERE crId, then UPDATE feedbackItems SET resolvedInVersionId = version.id WHERE id IN (feedbackIds) -- only if feedbackIds.length > 0
    6. Insert workflowTransitions: entityType 'change_request', fromState 'approved', toState 'merged', actorId, metadata: { event: 'MERGE', mergeSummary, versionLabel }
    7. Return { cr (updated), version }

    Add a helper function `getNextVersionLabel(tx, documentId)` that:
    - Queries documentVersions for the given document, ordered by createdAt desc, limit 1
    - If no row, return 'v0.1'
    - Parse the last label: extract number after 'v0.' and increment
    - Return formatted string 'v0.{N+1}'
