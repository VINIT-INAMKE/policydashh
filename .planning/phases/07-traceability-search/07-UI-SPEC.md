**Do NOT use accent for:** status badges (use existing Phase 4/5 semantic colors), the "Export PDF" button (use outline variant — secondary to CSV), navigation items, matrix chain cell borders, section or stakeholder name labels.

### Tab: Search

**Layout:** Single column. Top: search input (full-width, prominent). Below: tabbed search scope selector. Below scope tabs: results list.

**Focal point declaration:** Eye lands on the search input first. After a query is entered, eye moves to the scope tabs (Feedback / Policy Content / Change Requests), then to the results list below.

| Element | Component | Variant | Notes |
|---------|-----------|---------|-------|
| Search input | `Input` | -- | Placeholder: "Search feedback, policy content, or change requests…", Search icon prefix (via input-group), full-width on page, max-width 640px centered |
| Scope tabs | `Tabs` | -- | 3 tabs: "Feedback ({N})" / "Policy Content ({N})" / "Change Requests ({N})". Count shown after results load. |
| Result card (Feedback) | `Card` | default | FB-NNN badge + title + matched excerpt (highlighted substring) + section name + outcome badge |
| Result card (Policy Content) | `Card` | default | Section title + document title + matched excerpt (highlighted substring) + version label |
| Result card (CR) | `Card` | default | CR-NNN badge + title + status badge + matched excerpt |
| Match highlight | Custom `<mark>` | -- | `bg-primary/10`, no text-decoration, rounded-sm, px-xs. Applied to matched substring within excerpt. |
| Result count | `<span>` | Label (12px/400), muted-foreground | "{N} results" above results list |
| Empty state (no query) | Custom div | -- | Search icon (48px, muted-foreground), "Search across all feedback, policy sections, and change requests." |
| Empty state (no results) | Custom div | -- | SearchX icon (48px, muted-foreground), "No results for "{query}". Try different keywords." |
| Loading state | `Skeleton` | -- | 4 skeleton result cards |

**Search behavior:**
- Debounce: 400ms after last keystroke before firing query
- Minimum query length: 2 characters (below 2 chars: show no-query empty state, no error)
- Backend: tRPC search router using PostgreSQL ILIKE for v1 (Phase 7 CONTEXT.md: "ILIKE for v1"); full-text tsvector can be added in a later phase
- Cross-entity search: single query searches all three scopes simultaneously; counts update per scope tab
- CR filter tab (SRCH-04): within the "Change Requests" scope tab, expose inline mini-filter: Status (Checkbox group) + Section (Select) + Linked Feedback (Select) — same filter set as Phase 5 CR filter panel, rendered inline above results rather than as a sidebar

**Excerpt display:** Extract up to 200 characters of surrounding text around the first match occurrence. Highlight matched substring with `<mark>` treatment above. Show "…" at truncation boundaries.

## Phase-Specific Notes

### What This Phase Does NOT Include

- **Audit trail viewer**: AUDIT-04 (auditor views full audit trail) ships in Phase 9. The traceability matrix shows the FB→CR→Version chain, not raw audit log entries.
- **Evidence pack export**: AUDIT-05/06 (structured ZIP export) ships in Phase 9.
- **Role-aware dashboard integration**: UX-01 through UX-07 ship in Phase 8. This phase creates the traceability views but does not integrate them into role dashboards.
- **Real-time search**: tsvector full-text search is listed as a post-v1 improvement. Phase 7 uses PostgreSQL ILIKE for v1.
- **Workshop-originated feedback tracing**: WS-03/04 (workshop-to-section and feedback-to-workshop links) ship in Phase 10. Matrix shows workshop-originated feedback as normal feedback rows without workshop attribution in this phase.

### Data Dependencies

All data required for the traceability chain already exists in the database from prior phases:
- `cr_feedback_links` — FB → CR links (Phase 5)
- `cr_section_links` — CR → Section links (Phase 5)
- `changeRequests.mergedVersionId` — CR → Version link (Phase 5)
- `feedback.resolvedInVersionId` — FB → Version link (Phase 5)
- `workflowTransitions` — decision rationale (Phase 4)
- `documentVersions` — version labels and snapshots (Phase 6)
