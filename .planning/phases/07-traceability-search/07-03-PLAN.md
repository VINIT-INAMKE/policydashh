**Step 1: Create `search-result-card.tsx`.**

A polymorphic card component that renders differently based on result type. Three variants:

1. **FeedbackResultCard**: FB-NNN badge (monospace, muted) + title + matched excerpt (with highlighted substring) + section name + StatusBadge. Click navigates to feedback detail or opens sheet.

2. **SectionResultCard**: Section title + document title (if available) + matched excerpt (highlighted) + version label (if available). Click navigates to section in document editor.

3. **CRResultCard**: CR-NNN badge (monospace, muted) + title + CRStatusBadge + matched excerpt (highlighted). Click navigates to `/policies/[id]/change-requests/[crId]`.

**Highlight logic**: Create a `HighlightedText` sub-component.
- Props: `{ text: string, query: string }`.
- Split text on case-insensitive match of query string.
- Wrap matched portions in `<mark className="bg-primary/10 rounded-sm px-0.5">`.
- If text is longer than 200 chars, extract a 200-char window around the first match occurrence, prepend/append "..." at truncation boundaries.

**Excerpt extraction**: Create a `getExcerpt(text: string, query: string, maxLen = 200): string` helper function.
- Find first occurrence of query (case-insensitive) in text.
- Extract `maxLen` chars centered on the match position.
- Add "..." at boundaries if truncated.

Props for each card variant:
- `FeedbackResultCard`: `{ result: SearchFeedbackResult, query: string, documentId: string }`
- `SectionResultCard`: `{ result: SearchSectionResult, query: string, documentId: string }`
- `CRResultCard`: `{ result: SearchCRResult, query: string, documentId: string }`

**Step 2: Create `search-view.tsx`.**

Per UI-SPEC Search tab spec:

**Layout**: Single column. Top: search input. Below: scope tabs. Below scope: results list.

**Search input**:
- Use shadcn InputGroup with Search icon (from lucide-react) as prefix.
- Placeholder: "Search feedback, policy content, or change requests..."
- `role="searchbox"`, `aria-label="Search feedback, policy content, and change requests"`.
- Max-width 640px, centered.
- Controlled input: `useState('')` for raw query.
- Debounce: `const [debouncedQuery] = useDebounce(query, 400)` from `use-debounce` library.
- When query < 2 chars: show no-query empty state.
- When query in flight: show Loader2 spinner as suffix in input (via InputGroup).

**Three tRPC queries fired in parallel** (all enabled when debouncedQuery.length >= 2):
```typescript
const feedbackResults = trpc.traceability.searchFeedback.useQuery(
  { documentId, query: debouncedQuery },
  { enabled: debouncedQuery.length >= 2 }
)
const sectionResults = trpc.traceability.searchSections.useQuery(
  { documentId, query: debouncedQuery },
  { enabled: debouncedQuery.length >= 2 }
)
const crResults = trpc.traceability.searchCRs.useQuery(
  { documentId, query: debouncedQuery },
  { enabled: debouncedQuery.length >= 2 }
)
```

**Scope tabs** (shadcn Tabs, nested inside the outer traceability tabs):
- "Feedback ({N})" — count from feedbackResults.data?.length
- "Policy Content ({N})" — count from sectionResults.data?.length
- "Change Requests ({N})" — count from crResults.data?.length
- Counts update as each query resolves. Show "..." while loading.
- Default to "Feedback" tab.

**Results area per active scope tab**:
- Feedback: render FeedbackResultCard for each result.
- Policy Content: render SectionResultCard for each result.
- Change Requests: render CRResultCard for each result.

**CR scope inline filters** (SRCH-04): Within the "Change Requests" scope tab, render inline mini-filters ABOVE the results:
- Status: Checkbox group (Drafting, In Review, Approved, Merged, Closed) — filter crResults client-side.
- Section: Select dropdown — pass sectionId to the searchCRs query.
- These are inline (horizontal flex row), not a sidebar.
- Use useState for CR filter state. Re-query `trpc.traceability.searchCRs` when sectionId changes (add sectionId to the query input). Status filter is client-side on the returned results.

**Result count**: `<span>` above results: "{N} results" in 12px/400 muted-foreground.

**Empty states**:
- No query entered: Search icon (48px, muted-foreground), "Search across all feedback, policy sections, and change requests."
- No results: SearchX icon (48px, muted-foreground), 'No results for "{query}". Try different keywords.'
- Error: border-destructive on input, "Search is temporarily unavailable. Please try again." below input.

**Loading**: 4 skeleton result cards: header skeleton + excerpt skeleton (2 lines).

**aria-live**: Results list container: `aria-live="polite"` to announce result count changes for screen readers.

**Props**: `{ documentId: string }`.
