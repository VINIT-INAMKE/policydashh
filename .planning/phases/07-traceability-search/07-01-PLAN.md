**Step 3: Create `src/server/routers/traceability.ts` — the main new router.**

Implements 6 procedures:

1. **`matrix`** (TRACE-01, TRACE-02, TRACE-03): `requirePermission('trace:read')`
   - Input: `{ documentId: uuid, sectionId?: uuid, orgType?: enum, decisionOutcome?: enum, versionFromLabel?: string, versionToLabel?: string }`
   - Query: Multi-table JOIN:
     ```
     feedbackItems
       LEFT JOIN crFeedbackLinks ON feedbackItems.id = crFeedbackLinks.feedbackId
       LEFT JOIN changeRequests ON crFeedbackLinks.crId = changeRequests.id
       LEFT JOIN documentVersions ON changeRequests.mergedVersionId = documentVersions.id
       LEFT JOIN crSectionLinks ON changeRequests.id = crSectionLinks.crId
       LEFT JOIN policySections ON crSectionLinks.sectionId = policySections.id
       LEFT JOIN users ON feedbackItems.submitterId = users.id
     ```
   - Use LEFT JOIN (not INNER JOIN) for feedback-to-CR so unlinked feedback still appears in matrix (rows where CR is null show "--").
   - Dynamic conditions array: always filter by `eq(feedbackItems.documentId, input.documentId)`. Conditionally add: `eq(users.orgType, input.orgType)`, `eq(feedbackItems.sectionId, input.sectionId)` (NOT policySections — filter by feedback's section assignment), `eq(feedbackItems.status, input.decisionOutcome)`.
   - For version range filter: if versionFromLabel or versionToLabel provided, use a subquery to get version IDs for that label range, then filter `inArray(documentVersions.id, versionIds)` or `gte/lte` on documentVersions.createdAt.
   - Select shape: feedbackId, feedbackReadableId, feedbackTitle, feedbackStatus, feedbackDecisionRationale, feedbackIsAnonymous, submitterName, submitterOrgType, crId, crReadableId, crTitle, crStatus, sectionId, sectionTitle, versionId, versionLabel, feedbackCreatedAt.
   - **Anonymity enforcement**: Replicate the pattern from feedback.list — null out submitterName/submitterOrgType when isAnonymous=true unless ctx.user.role is admin or policy_lead.
   - **Cartesian product mitigation**: The JOIN of crSectionLinks x crFeedbackLinks through changeRequests produces N x M rows. Client will group by feedbackId. Alternatively, group server-side by collecting unique (feedbackId, crId) pairs and aggregating section titles. Use client-side grouping for simplicity (return flat rows, let UI deduplicate by feedbackId for display).
   - Add `orderBy(desc(feedbackItems.createdAt))`.
   - Add `.limit(200)` for safety; include `limit` and `offset` input params with defaults.

2. **`sectionChain`** (TRACE-04): `requirePermission('trace:read')`
   - Input: `{ sectionId: uuid }`
   - Query: Reverse join from policySections through crSectionLinks -> changeRequests -> crFeedbackLinks -> feedbackItems, LEFT JOIN documentVersions.
   - Select: sectionId, sectionTitle, crId, crReadableId, crTitle, crStatus, feedbackId, feedbackReadableId, feedbackTitle, feedbackStatus, feedbackDecisionRationale, versionId, versionLabel, mergedAt (documentVersions.createdAt).
   - Group results by version transition (client-side) — return flat rows ordered by documentVersions.createdAt DESC.

3. **`stakeholderOutcomes`** (TRACE-05): `requirePermission('feedback:read_own')`
   - Input: `{ documentId?: uuid }` — optional filter by document. For Policy Lead/Admin, add `userId?: uuid` to view another user's outcomes.
   - Query:
     ```
     feedbackItems
       LEFT JOIN documentVersions ON feedbackItems.resolvedInVersionId = documentVersions.id
       LEFT JOIN policySections ON feedbackItems.sectionId = policySections.id
     ```
   - WHERE: `eq(feedbackItems.submitterId, targetUserId)` — where targetUserId is ctx.user.id for stakeholders, or input.userId for admin/policy_lead (after checking `ctx.user.role in ['admin', 'policy_lead']`).
   - Select: feedbackId, readableId, title, status, decisionRationale, resolvedInVersionId, versionLabel, sectionId, sectionTitle, createdAt.
   - Order by createdAt DESC.

4. **`searchFeedback`** (SRCH-02): `requirePermission('feedback:read_all')`
   - Input: `{ documentId: uuid, query: string.min(2).max(200) }`
   - Query: `from(feedbackItems).leftJoin(users, ...).leftJoin(policySections, eq(feedbackItems.sectionId, policySections.id))` with `where(and(eq(documentId), or(ilike(title, term), ilike(body, term))))` where `term = '%' + input.query + '%'`.
   - Select: id, readableId, title, body (truncated client-side), status, sectionTitle, feedbackType, createdAt.
   - **Escape ILIKE special characters**: Replace `%` and `_` in input.query with `\%` and `\_` before wrapping in `%...%`.
   - Limit 50.

5. **`searchSections`** (SRCH-03): `requirePermission('document:read')`
   - Input: `{ documentId: uuid, query: string.min(2).max(200) }`
   - Query: `from(policySections).leftJoin(policyDocuments, ...) ` with `where(and(eq(documentId), ilike(policySections.title, term)))`.
   - Select: id, title, orderIndex, documentId.
   - SRCH-03 searches section TITLES only for v1 (JSONB ILIKE on content is unreliable — produces false positives on JSON syntax tokens).
   - Limit 50.

6. **`searchCRs`** (SRCH-04): `requirePermission('cr:read')`
   - Input: `{ documentId: uuid, query: string.min(2).max(200), status?: enum, sectionId?: uuid, feedbackId?: uuid }`
   - Query: `from(changeRequests)` with `where(and(eq(documentId), or(ilike(title, term), ilike(description, term))))`.
   - If sectionId: sub-select CR IDs from crSectionLinks where sectionId matches, then `inArray(changeRequests.id, crIds)`.
   - If feedbackId: sub-select CR IDs from crFeedbackLinks where feedbackId matches, then `inArray(changeRequests.id, crIds)`.
   - Select: id, readableId, title, description (truncated), status, createdAt.
   - Limit 50.
