## isPublicDraft flag (schema)

| Option | Description | Selected |
|--------|-------------|----------|
| New boolean column on `policy_documents` | Add `isPublicDraft BOOLEAN` directly to the documents table; simplest admin toggle | ✓ |
| Derive from another state field | Use an existing workflow field to signal public draft | |
| Tag on `documentVersions` row | Flag individual versions rather than the document | |

**Selected:** New boolean column on `policy_documents`
**Notes:** REQUIREMENTS.md PUB-07 explicitly says "documents tagged `isPublicDraft: true`", so a document-level flag is the requirement-driven choice. Default `false` keeps existing rows safe. Expose via a thin tRPC mutation `document.setPublicDraft` guarded by `policy:manage`.

## Framework content source

**Notes:** Framework page is a consultation surface, so it needs the latest draft, not published. Version snapshot is preferred because it locks structure for a consultation window; live fallback covers pre-version documents.
