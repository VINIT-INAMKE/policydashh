## Spacing Scale

Declared values (multiples of 4, 8-point grid, matching portal canonical pattern):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap (e.g. lucide icon + badge text) |
| sm | 8px | Compact: badge pill padding x/y, nav link padding y |
| md | 16px | Default: element spacing, section padding x, card padding |
| lg | 24px | Section gap, TOC aside padding, log entry spacing |
| xl | 32px | Layout column gap (content + nav: `gap-8`) |
| 2xl | 48px | Major section breaks (research page `<section>` top margin) |
| 3xl | 64px | Page top/bottom padding (`py-16` = 64px) |

Exceptions:
- Sticky nav top offset: `top-6` (24px) — matches `PublicSectionNav` existing code
- HR dividers between sections: `my-12` (48px) — matches `PublicPolicyContent` existing code
- Badge pill: `px-2 py-0.5` (8px / 2px) — matches `VersionStatusBadge` existing code

Source: Existing portal components verified in code; no new exceptions added.

## Color

All values reference existing CSS custom properties from `app/globals.css`. Phase 21 owns theme changes; this phase inherits as-is.

| Role | CSS Var | Usage |
|------|---------|-------|
| Dominant (60%) | `--background` = `oklch(1 0 0)` (white) | Page background, content area surfaces |
| Secondary (30%) | `--muted` = `oklch(0.97 0 0)` (near-white) | TOC sidebar (`bg-muted/50`), badge pills |
| Accent (10%) | `--primary` = `oklch(0.205 0 0)` (near-black) | Validated badge background (indigo: see badge spec below), active nav link bg, download CTA button |

Accent reserved for:
1. Download PDF button (primary Button variant)
2. Active section nav link (`bg-background font-medium text-foreground`)
3. SectionStatusBadge "Validated" state foreground (`--status-cr-merged-text`)
4. Join Consultation CTA button (primary Button variant on `/research`)

Source: `globals.css` verified. Accent is not used on hover states, borders, or secondary badges.

### SectionStatusBadge Color Mapping

Three variants cloned from `VersionStatusBadge` pattern using existing `--status-cr-*` vars:

| Variant | Background CSS Var | Text CSS Var | Icon |
|---------|-------------------|--------------|------|
| Draft | `--muted` (`oklch(0.97 0 0)`) | `--muted-foreground` | none |
| Under Review | `--status-cr-approved-bg` (`oklch(0.9 0.08 145)` green-tinted) | `--status-cr-approved-text` (`oklch(0.4 0.12 145)`) | `Clock` (lucide, size-3.5) |
| Validated | `--status-cr-merged-bg` (`oklch(0.88 0.07 265)` indigo) | `--status-cr-merged-text` (`oklch(0.4 0.12 265)`) | `CheckCircle2` (lucide, size-3.5) |

Badge pill shape: `inline-flex items-center gap-1 rounded-full px-2 py-0.5 text-xs font-medium`
Exact class parity with `VersionStatusBadge` and changelog page badge (line 82 in changelog/page.tsx).

Note: "Under Review" reuses `--status-cr-approved-bg/text` (green) because the approved CR vars are the closest semantic match available. If Phase 21 introduces amber vars, this mapping can be updated without contract breakage.

### PublicPolicyContent prop extension

`PublicPolicyContent` must accept an optional `sectionStatuses` prop of type `Map<string, 'draft' | 'under_review' | 'validated'>`. When provided, renders `SectionStatusBadge` inline after the `<h2>` section title. When absent, renders exactly as today (no badge). This keeps the portal page unaffected.

Proposed new signature:
```ts
interface PublicPolicyContentProps {
  sections: SectionSnapshot[]
  sectionStatuses?: Map<string, 'draft' | 'under_review' | 'validated'>
}
```

Badge placement: right of the `<h2>` title, same flex row, `flex items-center gap-2`.

## Component Specifications

### SectionStatusBadge

File: `app/(public)/framework/_components/section-status-badge.tsx`

```
type SectionStatus = 'draft' | 'under_review' | 'validated'

Props: { status: SectionStatus }
```

Rendering contract:

| status | pill classes | icon | text label |
|--------|-------------|------|------------|
| `draft` | `bg-muted text-muted-foreground` | none | "Draft" |
| `under_review` | `bg-[var(--status-cr-approved-bg)] text-[var(--status-cr-approved-text)]` | `Clock size-3.5` | "Under Review" |
| `validated` | `bg-[var(--status-cr-merged-bg)] text-[var(--status-cr-merged-text)]` | `CheckCircle2 size-3.5` | "Validated" |

Outer element: `<span className="inline-flex items-center gap-1 rounded-full px-2 py-0.5 text-xs font-medium" aria-label="{status label}">`

Accessibility: `aria-label` spells out full status word (e.g. "Under Review").

### WhatChangedLog

File: `app/(public)/framework/_components/what-changed-log.tsx`

Props:
```ts
interface WhatChangedLogProps {
  entries: Array<{
    sectionTitle: string      // resolved from sectionTitleMap — never raw sectionId
    mergeDate: string         // ISO date string, formatted as "MMM d, yyyy" via date-fns format()
    summary: string           // entry.summary from ChangelogEntry — ONLY this field; no CR IDs
  }>
}
```

Layout for each entry:

```
<div class="space-y-1 py-4">
  <div class="flex items-center gap-2">
    <span class="text-xs text-muted-foreground font-medium">{mergeDate}</span>
    <span class="text-xs text-muted-foreground">·</span>
    <span class="text-xs text-muted-foreground">{sectionTitle}</span>
  </div>
  <p class="text-sm text-foreground">{summary}</p>
</div>
<Separator />  {/* between entries, not after last */}
```

Empty state (no entries): icon `BookOpen size-12 text-muted-foreground` centered, heading "No changes merged yet", body "Updates will appear here as the framework evolves."

- `#key-themes`

- `#join-consultation`

TOC items:

## Workspace Toggle (isPublicDraft)

Minimal toggle placement: workspace policy detail page (`app/(workspace)/policies/[id]/page.tsx` or equivalent settings area). Visual spec:

```
<div class="flex items-center justify-between rounded-lg border p-4">
  <div class="space-y-0.5">
    <p class="text-sm font-medium">Public Draft Consultation</p>
    <p class="text-xs text-muted-foreground">Make this document visible on the public /framework page.</p>
  </div>
  <Switch checked={isPublicDraft} onCheckedChange={handleToggle} />
</div>
```

Component: shadcn `Switch` (already available in base-nova). On toggle: calls `document.setPublicDraft` tRPC mutation. No confirmation dialog (non-destructive toggle; reversible).
