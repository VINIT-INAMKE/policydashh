## Spacing Scale

Inherited from Phase 02 UI-SPEC (design system source of truth). No exceptions added for this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing |
| sm | 8px | Compact element spacing, card internal gaps, form field gaps |
| md | 16px | Default element spacing, card padding, list item padding |
| lg | 24px | Section padding, dialog body padding, group spacing |
| xl | 32px | Layout column gaps, page section margins |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom spacing |

Exceptions:
- File upload touch target: 44px min height for WCAG 2.5.5 Target Size compliance
- Workshop list card grid: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4` (matches policy list)
- Workshop detail two-column layout: left panel 320px fixed, right content flex-1

## Color

Uses shadcn base-nova neutral CSS variable palette (already established in globals.css). All values reference CSS custom properties for dark mode compatibility.

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant (60%) | `var(--background)` | Page background, content areas, dialog backgrounds |
| Secondary (30%) | `var(--muted)` | Cards, sidebar panels, table header rows, artifact list background |
| Accent (10%) | `var(--primary)` | See reserved list below |
| Destructive | `var(--destructive)` | Delete workshop button, delete confirmation action |

**Accent (`var(--primary)`) reserved for these specific elements only:**
1. "Create Workshop" primary CTA button (filled, default variant)
2. "Save Workshop" / "Update Workshop" submit button in create/edit form (filled, default variant)
3. "Attach Artifact" primary upload action button (filled, default variant)
4. Active tab indicator on workshop list Upcoming/Past tabs
5. Selected state highlight in section-link picker (checkbox + row background)
6. "View All Gaps" link on Research Lead dashboard widget (leading to /feedback/evidence-gaps)

**Do NOT use accent for:** artifact type badges (use muted/secondary), secondary form actions (outline variant), filter controls (outline/ghost variant), navigation hover states, evidence metadata labels.

**Additional semantic colors (inherited, already in globals.css):**

| Semantic | Variable | Usage |
|----------|----------|-------|
| Border | `var(--border)` | Card borders, table borders, dialog borders, picker borders |
| Muted foreground | `var(--muted-foreground)` | Timestamps, metadata values, placeholder text, uploader name |
| Ring | `var(--ring)` | Focus rings on all interactive elements |
| Status accepted bg/text | `var(--status-accepted-bg/text)` | Feedback "accepted" badges on evidence-gaps page (inherited from Phase 4) |
| Status rejected bg | `var(--status-priority-high-bg)` | High-priority badges on evidence-gaps page |

**Artifact type color treatment:** Use `Badge` with `variant="secondary"` for all artifact types (promo, recording, summary, attendance, other). No custom per-type colors — muted palette only.

## Component Inventory

### Page: Workshop List (`/workshops`)

**Layout:** Full-width within workspace layout. Header row with title + actions. Tabs for Upcoming / Past. Content area with workshop cards grid.

**Focal point declaration:**
- Tabs + card grid: eye lands first on the "Upcoming" tab (active, accent indicator), then scans the card grid. Each card's focal point is the workshop title (16px/600), then the date+time line (14px/400, muted-foreground with Calendar icon), then the duration badge, then the card footer action row.
- Empty state: eye lands on Calendar icon (48px, muted-foreground, centered), then reads "No workshops yet" heading, then body copy, then "Create Workshop" CTA.

| Element | Component | Variant | Notes |
|---------|-----------|---------|-------|
| Page heading | `<h1>` | Heading (20px/600) | "Workshops" |
| Create button | `Button` | default | "Create Workshop" with Plus icon (16px). Visible only to `workshop_moderator` and `admin` roles. |
| Upcoming / Past tabs | `Tabs` + `TabsList` + `TabsTrigger` | default | Two tabs: "Upcoming" (default active) and "Past". Tab switch updates tRPC query filter param. |
| Workshop card | `Card` | default | See card layout below |
| Card actions menu | `DropdownMenu` | -- | Trigger: MoreHorizontal icon button. Items: "Edit Workshop", separator, "Delete Workshop" (destructive text). Visible only to `workshop_moderator` and `admin`. |
| Empty state (no workshops) | Custom div | -- | Centered, Calendar icon (48px, muted-foreground), heading + body copy, "Create Workshop" CTA |
| Loading state | `Skeleton` | -- | 3 skeleton cards matching card dimensions |
| Upcoming badge | `Badge` | secondary | Duration in minutes: "{N} min". Shown in card header row. |
| Registration link | `Button` | ghost, size="sm" | ExternalLink icon + "Register". Shown only when `registrationLink` is set. Opens in new tab. |

**Workshop card layout:**
```
[CardHeader]
  Row: Workshop title (16px/600, left) ... DropdownMenu trigger (right, role-gated)
  Row: Calendar icon (14px, muted-foreground) + formatted date+time (14px/400, muted-foreground)
[CardContent]
  Description (14px/400, truncated 2 lines, muted-foreground)
[CardFooter]
  Row: Duration badge (left) ... "Register" ghost button (right, conditional)
```

**Date formatting:** Use `date-fns` `format(parseISO(scheduledAt), 'MMM d, yyyy · h:mm a')` — e.g., "Apr 12, 2026 · 2:00 PM". tRPC returns `scheduledAt` as ISO string; always call `parseISO()` before formatting.

### Page: Create Workshop (`/workshops/new`)

**Layout:** Single-column centered form page. Max-width 640px. Back link at top. Form card below.

| Element | Component | Variant | Notes |
|---------|-----------|---------|-------|
| Back link | `Button` | ghost, size="sm" | ArrowLeft icon + "Back to Workshops" |
| Page heading | `<h1>` | Heading (20px/600) | "Create Workshop" |
| Form card | `Card` | default | Contains all form fields |
| Title field | `Input` | -- | Label: "Title", placeholder: "e.g., Stakeholder Consultation Session 1", required, autofocus |
| Description field | `Textarea` | -- | Label: "Description", placeholder: "Brief overview of the workshop's agenda and objectives", rows: 4, optional |
| Date & time field | `<input type="datetime-local">` | -- | Label: "Date & Time", required. Styled to match shadcn Input (same border, padding, ring). No external date picker library. |
| Duration field | `Input` | -- | Label: "Duration (minutes)", type="number", min="1", placeholder: "e.g., 90", optional |
| Registration link field | `Input` | -- | Label: "Registration Link", type="url", placeholder: "https://...", optional. HTTPS-only validation with Zod. |

| Submit button | `Button` | default | "Create Workshop" — disabled until title and date are non-empty |
| Loading state | Submit button shows Loader2 (animate-spin) icon, text unchanged, disabled |

**Validation:**
- Title: required, min 1 char, max 200 chars
- Description: optional, max 2000 chars
- scheduledAt: required, ISO datetime string (from `<input type="datetime-local">` native value)
- durationMinutes: optional, positive integer
- registrationLink: optional, HTTPS URL (Zod `z.string().url()`)
- On submit: navigate to `/workshops/[id]`, show success toast

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | All interactive elements reachable via Tab. Section/feedback link picker items navigable with Arrow keys within ScrollArea. |
| Focus management | Dialogs and sheets auto-focus first input. On close, focus returns to trigger element. On artifact remove, focus moves to next artifact item or "Attach Artifact" button. |

Section unlink button: `aria-label="Unlink section {sectionTitle}"`. Evidence-gaps "Attach Evidence" button: `aria-label="Attach evidence to {feedbackId}: {feedbackTitle}"`. |

## Phase-Specific Notes

### What This Phase Does NOT Include

- **Workshop status/lifecycle state machine:** Workshops are simple CRUD entities — no XState machine, no lifecycle states (Draft/Published/Cancelled). That complexity is not in WS-01 through WS-05.
- **Workshop attendee management:** No attendee list or RSVP tracking. Attendance is handled via the "attendance" artifact type (an uploaded file).
- **Full evidence repository page:** EV-03 is the "claims without evidence" gap view. A global evidence repository browser is not in scope for this phase.
- **Public workshop visibility:** Workshops live in the workspace (auth-required). Public portal for workshop summaries was completed in Phase 9 via evidence pack export.
- **Bulk feedback linking:** Feedback link picker is one-at-a-time via checklist. No bulk CSV import.

### Design Consistency Notes

- Workshop list cards must visually match the pattern of Policy list cards (Phase 02) and CR list cards (Phase 05): same Card component, same footer row layout, same DropdownMenu for actions.
- The "Claims Without Evidence" table follows the exact visual pattern of the Traceability matrix table (Phase 07) and Audit trail table (Phase 09): same `Table` component, same column header style, same row hover treatment.
- Evidence metadata display rows (EV-04) follow the exact artifact item layout established in `evidence-list.tsx` from Phase 04.

### Date Input Styling

```
className="flex h-9 w-full rounded-md border border-input bg-transparent px-3 py-1 text-sm shadow-sm transition-colors file:border-0 file:bg-transparent file:text-sm file:font-medium placeholder:text-muted-foreground focus-visible:outline-none focus-visible:ring-1 focus-visible:ring-ring disabled:cursor-not-allowed disabled:opacity-50"
```
This matches the shadcn Input component's className exactly.
