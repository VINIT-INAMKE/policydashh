## Interaction States

### Milestone Anchoring Flow (Admin)

| State | Visual |
|-------|--------|
| `defining` | "Mark ready" button enabled (if all slots met). No anchor UI. |
| `ready` (post-markReady, pre-Inngest) | Status badge flips to green "Ready". No anchor UI yet. |
| `anchoring` (Inngest running) | Badge shows spinning `Loader2` + "Anchoring" in `bg-primary/10 text-primary`. "Retry Anchor" button appears (outline, enabled). No Cardanoscan link yet. |
| `anchored` | Badge shows Lock + "Anchored" in indigo. Retry button gone. Cardanoscan link + truncated txHash visible below header. |

## Accessibility

- All external links include `rel="noopener noreferrer"` and an `aria-label` that describes the destination.
- `VerifiedBadge` rendered as `<a>` must have visible focus ring (`outline-ring/50` from global base styles).
- `RetryAnchorButton` minimum touch target 44px height.
- `Loader2 animate-spin` icons: `aria-hidden="true"` on the icon, spinner state communicated via button label text change ("Retryingâ€¦").
- Reduced motion: `animate-spin` on `Loader2` is exempt from `prefers-reduced-motion` override (it is not a decorative animation; it communicates active server state). No scroll-driven animation in this phase.
