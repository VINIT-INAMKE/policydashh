## Spacing Scale

Declared values (all multiples of 4 — 8-point grid, 4px sub-step):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), chip inner padding |
| sm | 8px | Compact row padding (`p-2`), badge spacing |
| md | 16px | Default card padding, form field gap (`gap-4`) |
| lg | 24px | Section padding (`space-y-6`), card header |
| xl | 32px | Page-level top margin, major section break |
| 2xl | 48px | Dashboard stat-card row top margin on desktop |
| 3xl | 64px | Page max-width gutter on ≥1280px viewports |

Exceptions:
- Touch targets on lifecycle action buttons: minimum 44px height (accessibility — keyboard/pointer parity)
- StatCard minimum height: 96px (matches existing `stat-card.tsx` `min-h-[96px]`)
- Filter panel left rail: 240px fixed width on desktop; collapses to top-of-list on mobile (Claude's discretion)

## Page Shells

### `/research-manage` — List page

Layout: full-width page with sticky header row (title + Create button), then two-column body: 240px left rail (filter panel) + flex-1 table. On mobile: filter panel collapses above the table (not hidden).

Header row:
- `<h1>` "Research Items" at 20px semibold
- Button "Create Research Item" (accent, primary) — links to `/research-manage/new`; visible only to roles with `research:create` permission

Table columns (D-08 — from CONTEXT.md):
- ReadableID: `RI-NNN` in `font-mono` `text-xs` Badge, secondary variant — sortable
- Title: `text-sm` truncated to 1 line — sortable, links to `/research-manage/[id]`
- Type: formatted enum label (`text-xs text-muted-foreground`)
- Status: `ResearchStatusBadge` — sortable
- Author(s): comma-joined authors string; "Confidential" when `isAuthorAnonymous = true`
- Published/Updated: relative time (`text-xs text-muted-foreground`) — default sort desc (`updatedAt`)

Filter panel (D-08):
- Document: `Select` (single — list from `trpc.document.list`)
- Type: multi-checkbox (8 enum values — `researchItemTypeEnum`)
- Status: multi-checkbox (4 status values)
- Author: `Select` (users filtered to `research_lead | admin | policy_lead` roles)
- Client-side multi-filter pattern (Phase 4 precedent)

RBAC scope (from CONTEXT.md D-08):
- `research_lead`: sees own items only (server-enforced via `assertOwnershipOrBypass`)
- `admin | policy_lead`: sees all items

Empty state (no items): heading "No research items yet", body "Create a research item to attach citable evidence to a policy document.", CTA button "Create Research Item".

Empty state (filter produces no results): heading "No items match these filters", body "Try adjusting the type or status filters.", no CTA.

Loading state: 4 rows of `Skeleton` at `h-10` with matching column widths.

### `/research-manage/new` — Create page

Layout: single-column centered (max-w-2xl), Card container. Back-arrow link to `/research-manage`. Page `<h1>` "New Research Item".

Form structure (metadata fields — D-01 two-step replaced by single page):

**Section 1 — Core metadata** (always visible):
- Title: `Input` required, placeholder "e.g. NITI Aayog Digital Public Infrastructure Report 2024"
- Document: `Select` required — `trpc.document.list`
- Type (`itemType`): `Select` required — drives upload-mode branch (D-03)
- Status chip: read-only "Draft" badge (new items always start as draft)

**Section 2 — Citation metadata** (all optional):
- Authors: `Input` placeholder "Comma-separated names" — hidden when `isAuthorAnonymous = true`
- Anonymous author toggle: `Switch` with label "Hide author identity on public surfaces" + `AnonymousPreviewCard` below (D-05 live preview)
- Published Date: `Input type="date"`
- DOI: `Input` placeholder "e.g. 10.1000/xyz123" — plain text, no validation (Phase 26 Q10)
- Journal / Source: `Input` placeholder "e.g. Nature, Indian Journal of Law"
- Peer Reviewed: `Checkbox` + label
- Description: `Textarea` rows=4, placeholder "Abstract or summary"

**Section 3 — Attachment** (conditional on `itemType` — D-03):
- `media_coverage` | `legal_reference` → External URL input only (`Input type="url"` required); file input hidden
- All other types → file upload only; URL input hidden
  - Upload zone: file input (accept list matches D-04 allowlist: PDF, DOCX, DOC, CSV, XLSX, XLS; max 32 MB)
  - Upload state: "Uploading…" progress bar → "Uploaded name.pdf · 2.3 MB" (D-02 fire-on-select)

Footer: "Save Draft" button (accent) + "Cancel" link (ghost). No submit-for-review on create — that action is on the detail page.

Identical layout to create page. Prefilled with existing item data. Footer: "Save Changes" (accent) + "Cancel" (ghost, returns to detail page). "Save Changes" mutation: `trpc.research.update`. Same upload-mode branch logic.

### `/research-manage/[id]` — Detail page

Layout: two-column on desktop (flex-1 main + 320px right sidebar); single column stacked on mobile.

**Main column:**
- Page header: `<h1>` = item title (20px semibold) + `ResearchStatusBadge` inline
- ReadableID: `RI-NNN` monospace caption below title
- Metadata grid: 2-col `dl` list — Type, Document, Published Date, DOI (linked), Journal, Peer Reviewed (yes/no), Authors or "Confidential"
- Description: full text (14px, 1.5 line-height)
- Linked artifact: filename + size + download icon (if `artifactId` set) or URL link (if `externalUrl` set)
- Decision Log section (below metadata): `<h3>` "Decision Log" — `ResearchDecisionLog` component with transition list

**Right sidebar:**
- Lifecycle actions card (D-14 permission-derived):
  - `research_lead` (item owner, status=draft): "Submit for Review" button (primary)
  - `research_lead` (item owner, status=draft|pending_review): "Edit" link-button (ghost)
  - `admin | policy_lead` (status=pending_review): "Approve" button (primary) + "Reject" button (destructive, expands inline textarea for `rejectionReason`)
  - `admin | policy_lead` (status=published): "Retract" button (destructive, triggers Alert-Dialog confirmation with reason textarea)
  - Reject rationale: inline `Textarea` that appears below "Reject" button on click; "Submit Rejection" confirms; no separate dialog
  - Retract reason: Alert-Dialog (modal) with required `Textarea` — "Confirm Retract" destructive button
- Linked entities card (below lifecycle):
  - Linked Sections list with per-link `relevanceNote` inline editable (click-to-edit textarea, D-07)
  - "Link Sections" button → opens `SectionLinkPicker` dialog
  - Linked Versions list
  - "Link Versions" button → opens `VersionLinkPicker` dialog
  - Linked Feedback list
  - "Link Feedback" button → opens `FeedbackLinkPicker` dialog
  - Each link row: entity label + unlink icon button (Tooltip: "Remove link")

## Interaction Contract

### AnonymousPreviewCard (D-05)
- Renders immediately below the `isAuthorAnonymous` Switch on the edit form
- Toggles between `Authors: {names}` (when switch off) and `Source: Confidential` (when switch on) on each Switch change — no server round-trip
- Uses `Card` with `bg-muted` at 8px padding, 12px caption text, muted-foreground color
- Same `shouldHideAuthors(item)` helper used on the detail page author rendering

### Lifecycle action — Reject (inline expand)
- "Reject" button: destructive variant
- On click: button animates to "Reject" (stays pressed), a `Textarea` appears below with label "Rejection reason (required)", and a "Submit Rejection" button (destructive) + "Cancel" link
- Cancel collapses back to single button — no state mutation
- "Submit Rejection" disabled until textarea has ≥1 non-whitespace character
- On success: sonner toast "Research item rejected." + page data re-fetched via `utils.research.getById.invalidate`

### Lifecycle action — Retract (Alert-Dialog)
- "Retract" button: destructive variant in sidebar
- On click: Alert-Dialog opens with title, body, required `Textarea`, "Confirm Retract" (destructive) + "Cancel"
- "Confirm Retract" disabled until textarea has ≥1 non-whitespace character
- On success: toast "Research item retracted." + invalidate

### Link pickers (D-06 — three separate dialogs)
- Controlled: `open` + `onOpenChange` props — parent owns state
- All use `Checkbox` multi-select with document/section/version label
- Confirm button: "Link {N} selected" — disabled when selection is empty
- `Promise.allSettled` bulk-link → consolidated toast (Phase 21 pattern)
- On dialog close: selection state resets to empty

### relevanceNote inline edit (D-07)
- Each linked section row: static text of `relevanceNote` (or placeholder "Add a relevance note…" in muted-foreground)
- Click on text → `Textarea` appears in-place with "Save note" + "Cancel" ghost buttons
- "Save note" calls `trpc.research.linkSection` update (or a dedicated `updateRelevanceNote` if router exposes it — check before adding)
- Cancel restores original text — no mutation

### File upload (D-02)
- Fires `POST /api/upload` on file-select (not on form save)
- Progress: `Progress` component (0–100%) during PUT to R2
- On success: replace file input with static row "Uploaded {filename} · {size}" + "Remove" ghost button
- "Remove" clears `artifactId` state (orphan artifact accepted — Phase 26 Q precedent)
- On error: show `Alert` (variant="destructive") below the upload zone; file input resets

### Table sort
- Clicking column header toggles asc/desc — caret icon from lucide (ChevronUp / ChevronDown) inline with header text
- Only one sort active at a time
- Default: `updatedAt DESC`

### Mobile filter panel
- At <768px: filter panel renders as a collapsible `Collapsible` above the table with "Filters" trigger row + chevron icon
- Filter count badge on the trigger when any filter is active (e.g. "Filters · 2")

### Keyboard navigation within link pickers
- `Tab` moves between search input and checkbox list
- `Space` / `Enter` toggles focused checkbox
- `Escape` closes dialog (controlled — parent sets `open = false`)
