    Procedures (ALL use requirePermission, NONE use protectedProcedure alone for mutations):

    **list** - requirePermission('document:read').query
    - Select all from policyDocuments ordered by updatedAt desc
    - For each document, also return section count
    - Use a subquery or join to get section count per document
    - Return array of { id, title, description, createdAt, updatedAt, sectionCount }

    **getById** - requirePermission('document:read').input(z.object({ id: z.string().uuid() })).query
    - Fetch single document by id
    - Throw TRPCError NOT_FOUND if not found
    - Return document object

    **getSections** - requirePermission('document:read').input(z.object({ documentId: z.string().uuid() })).query
    - Select from policySections where documentId matches, ordered by orderIndex asc
    - Return array of sections

    **create** - requirePermission('document:create').input(z.object({ title: z.string().min(1).max(200), description: z.string().max(1000).optional() })).mutation
    - Insert into policyDocuments
    - writeAuditLog with action ACTIONS.DOCUMENT_CREATE, entityType 'document'
    - Return the created document

    **update** - requirePermission('document:update').input(z.object({ id: z.string().uuid(), title: z.string().min(1).max(200).optional(), description: z.string().max(1000).optional() })).mutation
    - Update policyDocuments set title/description/updatedAt where id matches
    - Throw TRPCError NOT_FOUND if no rows affected
    - writeAuditLog with action ACTIONS.DOCUMENT_UPDATE
    - Return updated document

    **delete** - requirePermission('document:delete').input(z.object({ id: z.string().uuid() })).mutation
    - Delete from policyDocuments where id matches (sections cascade automatically)
    - Throw TRPCError NOT_FOUND if no rows deleted
    - writeAuditLog with action ACTIONS.DOCUMENT_DELETE
    - Return { success: true }

    **createSection** - requirePermission('section:manage').input(z.object({ documentId: z.string().uuid(), title: z.string().min(1).max(200) })).mutation
    - Get max orderIndex for this document, new section gets max+1 (or 0 if no sections)
    - Insert into policySections with content default {} (empty Tiptap doc)
    - Also update the parent document's updatedAt timestamp
    - writeAuditLog with action ACTIONS.SECTION_CREATE
    - Return the created section

    **renameSection** - requirePermission('section:manage').input(z.object({ id: z.string().uuid(), documentId: z.string().uuid(), title: z.string().min(1).max(200) })).mutation
    - Update policySections set title, updatedAt where id matches AND documentId matches
    - Update parent document updatedAt
    - writeAuditLog with action ACTIONS.SECTION_RENAME
    - Return updated section

    **deleteSection** - requirePermission('section:manage').input(z.object({ id: z.string().uuid(), documentId: z.string().uuid() })).mutation
    - Delete from policySections where id AND documentId match
    - Update parent document updatedAt
    - writeAuditLog with action ACTIONS.SECTION_DELETE
    - Return { success: true }

    **reorderSections** - requirePermission('section:manage').input(z.object({ documentId: z.string().uuid(), orderedSectionIds: z.array(z.string().uuid()) })).mutation
    - CRITICAL: Neon HTTP driver may not support db.transaction(). Use sequential updates instead: loop through orderedSectionIds, update each section's orderIndex. Wrap in try/catch. If any update fails, the partial update is acceptable for MVP (sections have unique IDs, so re-fetching corrects the UI).
    - For each index i, update policySections set orderIndex = i, updatedAt = new Date() where id = orderedSectionIds[i] AND documentId matches
    - Update parent document updatedAt
    - writeAuditLog with action ACTIONS.SECTION_REORDER, payload includes orderedSectionIds
    - Return { success: true }

    **importDocument** - requirePermission('document:create').input(z.object({ title: z.string().min(1).max(200), description: z.string().max(1000).optional(), sections: z.array(z.object({ title: z.string().min(1).max(200), content: z.record(z.unknown()) })) })).mutation
    - Insert the document first, get its id
    - Insert all sections with documentId, orderIndex from array position
    - IMPORTANT: Since Neon HTTP may not support transactions, insert document first, then batch-insert sections. If sections fail, log the error but the orphaned document is acceptable for MVP.
    - writeAuditLog with action ACTIONS.DOCUMENT_IMPORT, payload includes sectionCount
    - Return { document, sections }
