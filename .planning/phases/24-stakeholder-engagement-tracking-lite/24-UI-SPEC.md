## New UI Surfaces

### 1. Inactive Users Widget (admin dashboard addition)

**Location:** `app/(workspace)/dashboard/_components/admin-dashboard.tsx` — inserted below the existing 4-StatCard grid row.

**Structure:**

```
[StatCard: N Inactive Users]    ← new stat in existing grid OR standalone above table

[Card: Inactive Users]
  CardHeader:
    "Inactive Users"  [Select: 30d ▾]
  CardContent:
    Table:
      Columns: Name | Role | Org Type | Last Active | Score | (actions)
      Rows: one per inactive user, sorted by Last Active ASC (oldest first) by default
      Sortable columns: Name, Last Active, Score (client-side sort)
      Action cell: "View Profile" Button size="sm" → /users/[id]
    Empty state (when zero inactive): icon + heading + body (see Copywriting)
    Loading state: 5× Skeleton h-12 w-full
```

**Inactivity window selector:**
- `Select` component positioned in `CardHeader` right slot
- Options: 7 days / 14 days / 30 days / 60 days / 90 days
- Default: 30 days
- Client-side filter — rerenders table without network call

**StatCard placement:** Add a 5th stat to the existing `grid-cols-2 lg:grid-cols-4` row; on large screens this wraps to a second row OR use a separate `grid-cols-1` row with a single `StatCard` above the inactive users card. Choose single-card row to avoid disrupting the existing 4-card layout — one `StatCard` in a `grid-cols-1` row immediately above the new inactive users `Card`.

### 2. Stakeholder Profile Page (new admin-only route)

**Location:** `app/(workspace)/users/[id]/page.tsx` (new file)

**Structure:**

```
Page layout: max-w-3xl mx-auto space-y-6

```

**Note:** Profile is read-only. No edit controls. Accessed via name-click in inactive widget or direct nav from users list.

## Interaction Contracts

### Sortable Table Columns (widget)

- Click column header to sort ascending; click again to sort descending.
- Sort icon: `ChevronUp` / `ChevronDown` from lucide-react, `size-4`, shown inline after column label, `text-muted-foreground` when inactive, `text-foreground` when active.
- Default sort: Last Active ASC (dormant users first).
- Sort state is local component state — no URL param, no server roundtrip.

### Inactivity Window Dropdown

- `Select` default value: `"30"` (representing 30 days).
- On change: refilter the existing users array client-side by comparing `lastActivityAt` to `(now - windowDays)`.
- No loading indicator needed — filter is synchronous over an already-fetched array.

### Name Link in Widget Table

- User name cell renders as a Next.js `Link` styled with `font-medium underline-offset-2 hover:underline` pointing to `/users/[id]`.
- Cursor: pointer.
- No external navigation — stays within the workspace shell.

### Empty States

- Icon: `UserX` from lucide-react, `size-10 text-muted-foreground`, centered.
- Heading: 16px/600, `text-foreground`.
- Body: 14px/400, `text-muted-foreground`.
- Vertical padding: `py-12` within CardContent.

### Loading Skeleton

- 5 rows of `Skeleton className="h-12 w-full rounded-lg"` inside `space-y-2`.
- Shown while `usersQuery.isLoading` is true.

## Accessibility

- All `<TableHead>` sortable columns include `aria-sort="ascending"` / `aria-sort="descending"` / `aria-sort="none"` on the active column.
- `StatCard` uses existing `aria-label={value + label}` pattern.

- Profile page uses semantic `<h1>` (user name, `text-xl font-semibold`) and `<h2>` (card titles, `text-xl font-semibold`) hierarchy.
- Empty state icons are `aria-hidden="true"`.
