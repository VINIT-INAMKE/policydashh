# Phase 9 — UI Design Contract

> Visual and interaction contract for Public Portal & Compliance. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Surface Summary

This phase builds **two distinct surfaces** with different layouts, auth requirements, and visual registers:

| Surface | Auth | Layout | Visual Register |
|---------|------|--------|----------------|
| **Public Portal** (`/portal`) | None required | Standalone public layout (no workspace nav) | Clean, editorial, read-only |
| **Audit Viewer** (`/audit`) | Required (auditor/admin role) | Workspace layout (existing header + nav) | Workspace-consistent, data-dense |

The public portal must feel categorically different from the workspace — a published government-facing document portal, not an internal tool. The audit viewer inherits all existing workspace patterns.

## Spacing Scale

Inherited from project baseline (source: 02-UI-SPEC.md). No changes for this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, icon-to-text spacing |
| sm | 8px | Compact element spacing, card internal gaps, audit table row gaps |
| md | 16px | Default element spacing, card padding, list item padding |
| lg | 24px | Section padding, dialog body padding, group spacing |
| xl | 32px | Layout column gaps, page section margins |
| 2xl | 48px | Major section breaks, public portal section vertical rhythm |
| 3xl | 64px | Page-level top/bottom spacing on public portal |

**Exceptions:**

- Public portal max content width: `max-w-3xl` (768px) — narrower reading column for policy text
- Public portal header height: 64px — matches comfortable nav height for public visitors
- Public portal section dividers: 48px vertical margin above/below `<hr>` elements
- Audit table row min-height: 44px — WCAG 2.5.5 target size for row actions
- Evidence pack export button: minimum touch target 44px height

## Typography

Inherited weights and scale. Public portal uses a slightly more editorial hierarchy with the existing 4-size scale.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Audit event metadata, table cells, consultation summary body text, portal section body |
| Label | 12px | 400 (regular) | 1.4 | Table column headers, audit timestamps, filter labels, version labels, badge text |
| Heading | 20px | 600 (semibold) | 1.2 | Public portal section headings, audit viewer card headings, page section titles |
| Display | 28px | 600 (semibold) | 1.2 | Public portal policy title, portal hero heading |

**Public portal body text exception:** Policy section content rendered from Tiptap JSON snapshot uses 16px/400/1.6 for comfortable long-form reading (editorial register, not workspace register). This is the only place 16px body appears in the application.

**Declared weights: 400 (regular) and 600 (semibold) only.** Mono font (`var(--font-geist-mono)`) used for: version labels (v0.1, v0.2), audit event IDs (8-char prefix), entity IDs in audit table.

## Color

Inherited from project CSS variable palette (source: globals.css). Uses oklch-based neutral scale.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `oklch(1 0 0)` white | `oklch(0.145 0 0)` near-black | Page background, content areas, public portal background |
| Secondary (30%) | `oklch(0.97 0 0)` near-white | `oklch(0.269 0 0)` dark zinc | Public portal sidebar/nav, audit table header rows, filter panel, card backgrounds |
| Accent (10%) | `oklch(0.205 0 0)` near-black | `oklch(0.922 0 0)` near-white | See reserved list below |
| Destructive | `oklch(0.577 0.245 27.325)` red | Same | No destructive actions in public portal. Audit viewer: none. |

**Accent reserved for these specific elements only:**

1. "Download PDF" primary CTA button on public portal (filled, primary variant)
2. "Export Evidence Pack" primary action button on audit viewer (filled, primary variant)
3. Active filter chip/badge in audit filter panel (selected state)
4. Public portal policy version selector active state (selected version label)
5. Public changelog version label pill (published version badge)

**Do NOT use accent for:** portal section headings, audit event badges (use muted), navigation links (use foreground), table sort indicators (use muted-foreground), consultation summary labels.

**Additional semantic colors (inherited):**

| Semantic | CSS Variable | Usage |
|----------|-------------|-------|
| Published version | `--status-cr-merged-bg` / `--status-cr-merged-text` | "Published · Immutable" badge — reuse VersionStatusBadge component |
| Border | `hsl(var(--border))` | All card borders, table borders, filter panel separators |
| Muted foreground | `hsl(var(--muted-foreground))` | Timestamps, secondary metadata, audit actor names, placeholder text |
| Ring | `hsl(var(--ring))` | Focus rings on all interactive elements |

**Public portal specific — sanitized privacy treatment:**

- Consultation summaries: no color-coded identity markers. Aggregate counts only (e.g., "14 submissions from Industry stakeholders"). Use `text-muted-foreground` for all attribution text.
- Never render stakeholder names, emails, or org-specific identifiers unless `privacyPreference = 'named'` and it is explicitly opted in.

|------------|----------|

## Phase-Specific Notes

### What This Phase Does NOT Include

- **Workshop evidence in ZIP**: Workshop data is Phase 10. Evidence pack ZIP includes a placeholder `workshop-evidence/README.txt` noting "Workshop evidence will be available after Phase 10 implementation."
- **Real-time audit updates**: Audit trail is a static paginated query. No WebSocket/polling — Auditor refreshes manually.
- **Public commenting**: Out of scope (REQUIREMENTS.md "Out of Scope" table — anonymous public commenting destroyed).
- **Stakeholder-facing consultation summary**: The `/portal/[policyId]/consultation-summary` page is publicly accessible but shows only sanitized aggregate data. No individual submission details visible anywhere on the public portal.
- **Audit log for public PDF downloads**: Public PDF downloads do not write audit entries (no authenticated actor).

### Navigation Integration

**WorkspaceNav update:** Add "Audit" link for auditor/admin roles only. Requires server-side role check to conditionally include nav item. Pattern: fetch current user role in `WorkspaceLayout`, pass as prop to `WorkspaceNav`, conditionally render "Audit" link.

### Public Portal: No Clerk Components

The `(public)` route group layout must NOT import or render any Clerk components (`UserButton`, `ClerkProvider`, `auth()`). It is a zero-auth surface. The "Internal Login" link in the footer is a plain `next/link` to `/sign-in`.
