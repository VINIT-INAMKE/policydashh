## Color

Inherited from Phase 2 and Phase 5. CSS custom properties from `app/globals.css` (oklch-based neutral palette).

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `oklch(1 0 0)` (white) | `oklch(0.145 0 0)` (near-black) | Page background, diff view background, dialog backgrounds |

| Destructive | `oklch(0.577 0.245 27.325)` (red) | `oklch(0.704 0.191 22.216)` (red) | Not used for primary actions in this phase; reserved for any future delete-version flow |

All color values referenced via CSS custom properties only. Never hardcode oklch values directly at component call sites.

**Accent reserved for these specific elements only:**
1. "Publish Version" primary CTA button (filled) on the publish confirmation dialog
2. "Compare Versions" submit button (filled) on the version comparison selector
3. Active version item highlight in the version history list (left border: 2px `var(--primary)`)
4. Version comparison selector — selected version indicator badge
5. Focus ring on interactive elements (inherited from base-nova ring behavior)

**Do NOT use accent for:** version status badges (use semantic colors below), diff addition/removal highlights (use semantic green/red below), changelog entries, secondary actions (use outline variant), navigation items.

**Additional semantic colors for versioning:**

| Semantic | CSS Variable / Value | Usage |
|----------|---------------------|-------|
| Version status: Draft | `var(--muted)` bg + `var(--muted-foreground)` text | Badge; version exists but not published |
| Version status: Published | `--status-cr-approved-bg` bg + `--status-cr-approved-text` text | Reuse green variables from Phase 5; published = positive terminal state |
| Diff: added line | `oklch(0.94 0.06 145)` bg + `oklch(0.35 0.12 145)` text | Green tint for added content in diff view |

| Immutable indicator | `--status-cr-merged-bg` bg + `--status-cr-merged-text` text | Reuse indigo from Phase 5; published+immutable badge uses indigo |

**Implementation note:** Declare `--diff-added-bg`, `--diff-added-text`, `--diff-removed-bg`, `--diff-removed-text` as CSS variables in `app/globals.css`. Do not hardcode oklch values at component call sites.

### Component: SectionDiffView

Rendered below the VersionComparisonSelector after "Compare Versions" is clicked.

**Layout:** Side-by-side two-column diff on desktop (>= 1024px). Single-column inline diff on mobile (< 1024px). Each column has a sticky header showing the version label.

| Element | Component | Variant | Notes |
|---------|-----------|---------|-------|
| Diff container | Custom div | -- | border, rounded-md, overflow-hidden. max-height: 60vh with scroll. |
| Left column header | Custom div | -- | bg-muted, border-b, padding sm. Version label (monospace, 12px) + "Base" label (12px, muted-foreground). |
| Right column header | Custom div | -- | bg-muted, border-b, padding sm. Version label (monospace, 12px) + "Target" label (12px, muted-foreground). |
| Line: unchanged | Custom row | -- | bg-background. Line number (12px, monospace, muted-foreground, w-8) + content (14px/400). |
| Line: removed (left only) | Custom row | -- | bg `var(--diff-removed-bg)`, text `var(--diff-removed-text)`. Minus prefix in gutter. |
| Line: added (right only) | Custom row | -- | bg `var(--diff-added-bg)`, text `var(--diff-added-text)`. Plus prefix in gutter. |
| No content state | Custom div | -- | Centered text: "This section had no content in version {v0.N}." (14px/400, muted-foreground) |
| Identical content state | Custom div | -- | Centered text: "No changes in this section between {v0.X} and {v0.Y}." (14px/400, muted-foreground) |
| Loading state | `Skeleton` | -- | 8 skeleton rows (h-5 w-full) with 4px gaps inside each column |

**Diff algorithm:** Compare stored `sectionsSnapshot` JSONB field on `document_versions`. Extract plain text per block for line-level diff using a simple LCS diff (e.g., `diff` npm package). Render as line-level unified diff in side-by-side columns. Block structure context (headings, paragraphs) is indicated by a subtle type label (12px, muted-foreground) at the start of each block group.

**Side-by-side layout (desktop):**
```
[Left column — Base version v0.1]  |  [Right column — Target version v0.2]
  line 1: unchanged                       line 1: unchanged
- line 2: removed text                  + line 2: added text (replacement)
  line 3: unchanged                       line 3: unchanged
```

**Inline diff (mobile < 1024px):** Render both removed and added lines in a single column, with color coding. Removed lines first (red), then added lines (green), grouped per changed block.

## Phase-Specific Notes

### What This Phase Does NOT Include

- **Public portal view**: VER-06 states the publish action makes the version visible on the public portal (Phase 9). This phase implements the publish action and immutability, but the public-facing read-only page ships in Phase 9.
- **Traceability matrix**: The full feedback→CR→section→version chain view ships in Phase 7. The version detail shows direct CR and FB links inline only.
- **PDF export**: Phase 9 delivers PDF export of published versions (PUB-04). No export action in this phase.
- **Version deletion**: Not permitted — versions are append-only. No delete action in version history.

### Version Label Format

Version labels follow `v{major}.{minor}` pattern: v0.1, v0.2, v0.3. The `getNextVersionLabel` service (implemented in Phase 5) drives this; no UI computation needed. Display always as monospace `var(--font-geist-mono)`.

### Published + Immutable State

Once a version is published (Admin/Policy Lead clicks "Publish Version"):
- `publishedAt` timestamp is set on `document_versions` record
- `isPublished` flag is set to true
- No write operations are permitted against this version's `sectionsSnapshot` or `changelog` fields
- The UI must not render any edit affordances on published versions
- Both the status badge ("Published · Immutable" with Lock icon) and the lock indicator text below the changelog communicate immutability

```css
--diff-removed-text: oklch(0.45 0.12 27);
```

Dark mode variants must be declared in the `.dark` block with the same variable names. Use lower-lightness values for dark mode backgrounds (e.g., `oklch(0.25 0.07 145)` for added-bg dark).

### Diff Implementation Note for Executor

The diff comparison reads from `document_versions.sectionsSnapshot` (JSONB), not from live `policy_sections.content`. Extract plain text from Tiptap JSON per block using a traversal function. Apply a character-level or line-level LCS diff. The `diff` npm package (already a common transitive dependency) provides `diffLines()` for line-level diffs suitable for this use case. Render results with the color-coded row system defined in `SectionDiffView`.
