| Property | Value | Source |

## Spacing Scale

Declared values (multiples of 4, 8-point base):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding |
| sm | 8px | Compact stacks within cards, icon-to-text gap |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Card gap in workshop grid, section padding |
| xl | 32px | Page section breaks (Upcoming → Live → Past) |
| 2xl | 48px | Page top padding on `/workshops` (matches `/participate` `pt-12`) |
| 3xl | 64px | Reserved — not used in Phase 20 surfaces |

Exceptions:
- Touch target for the "Register" button: minimum 44px height (accessibility — interactive primary CTA)
- Star rating input: each star touch target 44×44px minimum
- Modal embed container: `max-w-2xl` wide, height constrained to `80vh` with internal scroll to accommodate cal.com iframe

## Color

All values sourced from `app/globals.css` `.cl-landing` scope and shadcn `:root` CSS variables. Phase 20 public surfaces stay within `.cl-landing` scope. Admin surface stays within default shadcn scope.

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `--cl-surface: #f7fafc` | `/workshops` page background, feedback form page background |
| Secondary (30%) | `--cl-surface-container: #ebeef0` / `--card: oklch(1 0 0)` | Workshop cards background, modal popover background |
| Accent (10%) | `--cl-primary: #000a1e` | Register CTA button background, Submit Feedback button background, star-fill active color |

- Spots-left badge when low (≤3 spots): `--status-rejected-bg: oklch(0.95 0.04 27)` background with `text-destructive` — signals urgency without introducing new tokens.
- Spots-left badge when normal (>3 spots): `--secondary` background (`oklch(0.97 0 0)`) with `--secondary-foreground` text.

### Surface A: Public `/workshops` Listing Page

**Route:** `/workshops` (server-rendered, no auth)
**Scope class:** `.cl-landing` (matches Phase 19 `/participate`)

#### Layout

```
<div class="cl-landing min-h-screen bg-[var(--cl-surface)]">
  <main class="mx-auto max-w-3xl px-4 pt-12 pb-16 sm:px-6 sm:pt-16">
    <header> <!-- page title + subtitle --> </header>
    <section> <!-- Upcoming Workshops --> </section>
    <section> <!-- Live Workshops --> </section>
    <section> <!-- Past Workshops --> </section>
  </main>
</div>
```

Page max-width: `max-w-3xl` (wider than `/participate`'s `max-w-2xl` to accommodate the 3-column card grid at desktop).

#### Section Structure

Three ordered sections, each only rendered if it contains at least one workshop. If a section is empty, it is omitted entirely (do not render an empty section heading).

| Section | Filter | Card Variant |
|---------|--------|--------------|
| Upcoming Workshops | `status='upcoming' AND scheduled_at > now()` | Full card — title, date, description, spots-left badge, Register CTA |
| Live Workshops | `status='in_progress'` | Full card — title, "Live now" badge, description, Register CTA (no spots-left — in progress) |
| Past Workshops | `status='completed' AND scheduled_at < now()` | Compact card — title, date only; "View summary" link if approved summary artifact exists |

#### Section Heading

```
<h2 class="text-xl font-semibold leading-[1.2] text-[var(--cl-on-surface)] mb-4">
  Upcoming Workshops
</h2>
```

#### Card Grid

```
<div class="grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3">
  <!-- WorkshopPublicCard components -->
</div>
```

#### Upcoming/Live Workshop Card Anatomy

Uses `<Card>` from `@/components/ui/card` (shadcn base-nova). Content layout:

```
CardHeader
  ├── <h3> title — text-base font-semibold leading-snug
  └── date line — text-sm text-muted-foreground + Calendar icon (size-3.5)
CardContent
  ├── description — text-sm text-muted-foreground line-clamp-3
  └── spots-left badge (Upcoming only, when maxSeats is non-null)
CardFooter
  └── Register button — full-width, h-12, font-semibold
```

**Spots-left badge:** Rendered as `<Badge>` component.
- `maxSeats` is null: render nothing (open registration — no badge shown)
- `available > 3`: `<Badge variant="secondary">{available} spots left</Badge>`
- `available > 0 AND available <= 3`: `<Badge>` with `bg-[var(--status-rejected-bg)] text-destructive font-semibold` — "{available} spot{s} left"
- `available === 0`: `<Badge variant="secondary">Fully booked</Badge>` + disable Register button (cal.com handles the booking-side state, but the button should not open the modal when 0 available)

**Live badge:** Upcoming/Live distinction — Live cards show a green `<Badge>` with `bg-[var(--status-accepted-bg)] text-[var(--status-accepted-text)]` and text "Live now" in CardHeader adjacent to the title.

#### Past Workshop Card Anatomy

Compact variant — no Register button, no description, no spots badge:

```
CardHeader
  ├── <h3> title — text-base font-semibold leading-snug
  └── date — text-sm text-muted-foreground
CardFooter (only if approved summary exists)
  └── <a href="/portal/workshops/{id}/summary"> "View summary →" </a>
      text-sm font-medium text-[var(--cl-primary,#000a1e)] hover:underline
```

If no approved summary: CardFooter is omitted. Card renders title + date only.

#### Register Button (Upcoming/Live cards)

```
<Button
  class="h-12 w-full text-base font-semibold"
  style="background-color: var(--cl-primary, #000a1e); color: #ffffff;"
  onClick={openModal}
  aria-label="Register for {workshopTitle}"
>
  Register
</Button>
```

Disabled state (fully booked):
```
<Button disabled class="h-12 w-full text-base font-semibold opacity-50 cursor-not-allowed">
  Fully booked
</Button>
```

#### Cal.com Modal (Registration Embed)

Trigger: Register button click opens `<Dialog>` from `@/components/ui/dialog` (base-ui Dialog primitive).

Dialog content sizing: `max-w-2xl w-full` — wider than the default `sm:max-w-sm` in `DialogContent`. Override via className prop.

```
<DialogContent className="max-w-2xl p-0 overflow-hidden">
  <DialogHeader className="px-6 pt-5 pb-4 border-b">
    <DialogTitle>{workshopTitle}</DialogTitle>
    <DialogDescription>{formattedDate}</DialogDescription>
  </DialogHeader>

  <!-- Cal.com embed — lazy-loaded -->
  <div class="min-h-[400px] max-h-[70vh] overflow-y-auto">
    <Suspense fallback={<CalEmbedSkeleton />}>
      <LazyCalEmbed calLink={calcomLink} />
    </Suspense>
  </div>
</DialogContent>
```

Cal.com embed loading skeleton (shown while `@calcom/embed-react` chunk loads):

```
<div class="flex flex-col gap-3 p-6 animate-pulse">
  <Skeleton class="h-8 w-3/4 rounded-md" />
  <Skeleton class="h-4 w-1/2 rounded-md" />
  <Skeleton class="h-48 w-full rounded-md" />
  <Skeleton class="h-10 w-full rounded-md" />
</div>
```

Uses `<Skeleton>` from `@/components/ui/skeleton`.

Lazy-load pattern: `const LazyCalEmbed = dynamic(() => import('./_components/cal-embed'), { ssr: false })` — load only on Register click, not on page render.

#### Empty State — No Upcoming Workshops

When all three sections are empty (no workshops at all):

```
<div class="flex flex-col items-center gap-4 py-16 text-center">
  <CalendarX class="h-10 w-10 text-muted-foreground" />   <!-- lucide icon -->
  <p class="text-base font-semibold text-[var(--cl-on-surface)]">
    No workshops scheduled yet
  </p>
  <p class="text-sm text-muted-foreground max-w-sm">
    Check back soon — upcoming consultation sessions will appear here.
  </p>
</div>
```

When only specific sections are empty: those sections are simply omitted. No per-section empty state.

### Surface B: `/participate` Workshop Feedback Mode

**Route:** `/participate?workshopId={uuid}&token={jwt}` (public, no auth, no Turnstile)
**Scope class:** `.cl-landing` (identical to intake mode)

#### Mode Detection

Server component reads `searchParams.workshopId` and `searchParams.token`. If both present, validate JWT server-side. Three rendering outcomes:

| Condition | Rendered Surface |
|-----------|-----------------|
| `workshopId` absent | Existing intake form (Phase 19, unchanged) |
| `workshopId` present, token valid | Workshop feedback form (new) |
| `workshopId` present, token invalid or expired | Expired link landing |

#### Page Wrapper (identical to intake mode)

```
<div class="cl-landing min-h-screen bg-[var(--cl-surface)]">
  <main class="mx-auto max-w-2xl px-4 pt-12 pb-16 sm:px-6 sm:pt-16">
    <header class="mb-8 text-center sm:mb-12">
      <h1 class="text-[28px] font-semibold leading-[1.2] text-[var(--cl-on-surface)]"
          style="font-family: var(--font-cl-headline, Newsreader, serif)">
        Share Your Feedback
      </h1>
      <p class="mt-3 text-base leading-relaxed text-muted-foreground">
        Your insights from the workshop help shape the policy consultation.
      </p>
    </header>

    <WorkshopFeedbackForm workshopId={workshopId} token={token} />
  </main>
</div>
```

#### Feedback Form Card Anatomy

Uses `<Card class="px-6 py-8 sm:px-8">` matching Phase 19 form card exactly.

```
<Card class="px-6 py-8 sm:px-8">
  <form class="flex flex-col gap-6">
    <h2 class="text-xl font-semibold text-[var(--cl-on-surface)]">
      Workshop Feedback
    </h2>

    <!-- Error banner (same pattern as intake form) -->
    {topError && <div role="alert" class="rounded-md border border-destructive/30 bg-destructive/5 px-4 py-3 text-sm text-destructive">{topError}</div>}

    <!-- Rating Section -->
    <section class="flex flex-col gap-3">
      <Label>How would you rate this workshop?</Label>
      <StarRating value={rating} onChange={setRating} />
      {ratingError && <p class="text-sm text-destructive">{ratingError}</p>}
    </section>

    <Separator />

    <!-- Comment Section -->
    <section class="flex flex-col gap-3">
      <Label htmlFor="comment">
        Your feedback <span class="text-muted-foreground">(required)</span>
      </Label>
      <Textarea id="comment" rows={6} maxLength={4000} placeholder="Share what you found valuable, what could be improved, or any specific policy insights..." />
      <p class="text-right text-xs text-muted-foreground">{charCount}/4000</p>
      {commentError && <p class="text-sm text-destructive">{commentError}</p>}
    </section>

    <Separator />

    <!-- Section Selector (optional) -->
    <section class="flex flex-col gap-3">
      <Label htmlFor="sectionId">
        Which policy section does this relate to? <span class="text-muted-foreground">(optional)</span>
      </Label>
      <Select value={sectionId || null} onValueChange={...}>
        <SelectTrigger id="sectionId">
          <SelectValue placeholder="Select a section (optional)" />
        </SelectTrigger>
        <SelectContent>
          {sections.map(s => <SelectItem key={s.id} value={s.id}>{s.title}</SelectItem>)}
        </SelectContent>
      </Select>
    </section>

    <!-- Submit Button -->
    <Button
      type="submit"
      class="h-12 w-full text-base font-semibold"
      style="background-color: var(--cl-primary, #000a1e); color: #ffffff;"
    >
      {submitting ? 'Submitting…' : 'Submit Feedback'}
    </Button>

    <p class="text-center text-sm text-muted-foreground">
      Your feedback is submitted under your workshop registration and linked to this consultation.
    </p>
  </form>
</Card>
```

#### Star Rating Input Component

New component `StarRating` — built from lucide `Star` icons. No third-party library.

```
<div role="radiogroup" aria-label="Rating" class="flex gap-1">
  {[1,2,3,4,5].map(n => (
    <button
      key={n}
      type="button"
      role="radio"
      aria-checked={rating === n}
      aria-label={`${n} star${n > 1 ? 's' : ''}`}
      class="h-11 w-11 flex items-center justify-center rounded-md transition-colors
             hover:bg-muted focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring"
      onClick={() => setRating(n)}
    >
      <Star
        class="h-7 w-7 transition-colors"
        style={rating >= n
          ? { fill: 'var(--cl-primary,#000a1e)', color: 'var(--cl-primary,#000a1e)' }
          : { fill: 'none', color: 'var(--cl-outline,#74777f)' }
        }
      />
    </button>
  ))}
</div>
```

Touch target: 44×44px per star button (h-11 w-11 = 44px). Meets WCAG 2.5.5 minimum.
Keyboard: arrow keys move between stars (radiogroup pattern).

#### Success State (Post-Submit)

Replaces the form in-place (no redirect — matches Phase 19 `form-replace` pattern):

```
<Card role="status" aria-live="polite" class="flex flex-col items-center gap-4 px-6 py-12 text-center">
  <CheckCircle2 class="h-12 w-12" style="color: var(--cl-tertiary-fixed-dim, #66dd8b)" />
  <h2 class="text-xl font-semibold text-[var(--cl-on-surface)]">
    Feedback submitted.
  </h2>
  <p class="max-w-md text-base leading-relaxed text-muted-foreground">
    Thank you for sharing your insights. Your feedback is now part of the policy consultation record.
  </p>
</Card>
```

#### Expired / Invalid Token Landing

Rendered server-side when JWT validation fails. No form shown.

```
<Card class="flex flex-col items-center gap-4 px-6 py-12 text-center">
  <AlertCircle class="h-12 w-12 text-destructive" aria-hidden="true" />
  <h2 class="text-xl font-semibold text-destructive">
    This link has expired
  </h2>
  <p class="max-w-md text-base leading-relaxed text-muted-foreground">
    Feedback links are valid for 14 days after the workshop. This link is no longer active.
  </p>
  <p class="text-sm text-muted-foreground">
    If you believe this is an error, contact the workshop organizer.
  </p>
</Card>
```

### Surface C: Admin Workshop Create Form — `maxSeats` Field

**Route:** `/workshops/new` (authenticated, workspace scope)
**Scope:** Default shadcn workspace styles — no `.cl-landing`

One new field added to the existing `CreateWorkshopPage` form, inserted after "Duration (minutes)" and before "Registration Link":

```
<div class="flex flex-col gap-2">
  <Label htmlFor="workshop-maxseats">
    Maximum seats <span class="text-muted-foreground text-xs">(optional — leave blank for open registration)</span>
  </Label>
  <Input
    id="workshop-maxseats"
    type="number"
    min="1"
    max="10000"
    placeholder="e.g., 50"
    value={maxSeats}
    onChange={(e) => setMaxSeats(e.target.value)}
  />
</div>
```

No validation error state needed beyond browser's native `min="1"` constraint. If blank, `maxSeats` is passed as `undefined` to the tRPC mutation (null in DB = open registration).

## Accessibility Requirements

- All form fields have associated `<Label>` with matching `htmlFor`/`id` pair
- Star rating uses `role="radiogroup"` + individual `role="radio"` + `aria-checked` + `aria-label="N stars"`
- Star touch targets: 44×44px minimum (h-11 w-11)
- Register button `aria-label="Register for {workshopTitle}"` (title provides context beyond "Register")
- Empty state uses `role="status"` only if dynamically updating; static SSR empty state needs no role
- Success card: `role="status" aria-live="polite"` (same as Phase 19 `ParticipateSuccess`)
- Expired link card: `role="alert"` (error, assertive)
- Modal: `<DialogTitle>` always present (required for screen reader announcement)
- Spots-left badge: not interactive — pure `<span>` wrapped in `<Badge>`, no role needed
- Character counter below Textarea: `aria-live="polite"` on the count span so screen readers announce updates
