# Phase 21 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (base-nova style) |
| Preset | base-nova / neutral / CSS variables — `components.json` confirmed |

**Theme split:** This phase has TWO distinct theme contexts.

| Surface | Theme | Scope |

**Source of truth:** `app/globals.css` lines 361–468 (`.cl-landing` block). Do NOT invent new tokens. Map every public surface decision to an existing `--cl-*` variable.

## Spacing Scale

Standard 8-point scale — all values multiples of 4. Applies to both theme contexts.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip spacing |
| sm | 8px | Compact element internal padding (badge, tag) |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section padding, card internal padding |
| xl | 32px | Layout gaps between major card groups |
| 2xl | 48px | Page vertical breathing room between sections |
| 3xl | 64px | Hero / page-top top padding |

**Exceptions:**

- Mobile hamburger tap target: 44px minimum hit area (accessibility floor — wraps a 24px icon with `p-[10px]`)
- Public nav link active underline: 2px border-bottom (not a spacing token; visual only)
- Summary card divider between sections on `/portal`: `my-12` (`hr` separator — carried forward from `PublicPolicyContent` existing pattern)

## Typography

Two font families, four semantic roles. All sizes declared in px, applied via Tailwind arbitrary values matching existing codebase pattern (`text-[NNpx]`).

**Weight constraint: exactly 2 weights per surface — 400 (regular) and 600 (semibold). Weight 500 (medium) is not used anywhere in this phase.**

### Public shell + portal surfaces (`.cl-landing` context)

| Role | Size | Weight | Line Height | Font | CSS var |
|------|------|--------|-------------|------|---------|
| Display | 28px | 600 (semibold) | 1.2 | Newsreader | `--font-cl-headline` |
| Heading | 20px | 600 (semibold) | 1.2 | Newsreader | `--font-cl-headline` |
| Body | 16px | 400 (regular) | 1.8 | Inter | `--font-cl-body` |
| Label / nav | 12px | 600 (semibold) | 1.0 | Inter | `--font-cl-body` |

**Evidence:** `app/(public)/participate/page.tsx` line 45 uses `text-[28px] font-semibold leading-[1.2]` for h1. `public-policy-content.tsx` line 30 uses `text-[20px] font-semibold leading-[1.2]` for section h2. `text-[16px] font-normal leading-[1.8]` for body prose. Nav links in `app/page.tsx` use `text-xs uppercase tracking-widest font-semibold` (12px/600). The `tracking-widest uppercase` treatment provides sufficient visual differentiation at this size without a third weight.

Nav links: `text-xs font-semibold uppercase tracking-widest` — do NOT change to a different size for the new shell header.

### Workspace moderator review card (shadcn workspace context)

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Card heading | 16px | 600 (semibold) | 1.4 |
| Body / prose | 14px | 400 (regular) | 1.6 |
| Badge / label | 12px | 600 (semibold) | 1.0 |
| Textarea | 14px | 400 (regular) | 1.6 |

Use Tailwind defaults (`text-sm`, `text-base`, `font-semibold`) — no arbitrary values needed inside the workspace.

## Color

### Surface A — Public shell (`.cl-landing` palette)

| Role | Hex / var | Usage |
|------|-----------|-------|
| Dominant (60%) | `#f7fafc` — `var(--cl-surface)` | Page background, main content area |
| Secondary (30%) | `#ebeef0` — `var(--cl-surface-container)` | Nav background (glassmorphism: `bg-slate-50/80 backdrop-blur-md`), summary cards, placeholder cards |
| Accent (10%) | `#179d53` — `var(--cl-on-tertiary-container)` | Active nav underline, "Approved" summary badge background, "Participate" link in nav on hover |
| Text primary | `#000a1e` — `var(--cl-primary)` | All headings, logo wordmark |
| Text secondary | `#44474e` — `var(--cl-on-surface-variant)` | Body copy, nav links (inactive) |
| Destructive | `#ba1a1a` — `var(--cl-error)` | "Blocked" guardrail section badge, error state text |

**Accent reserved for exactly:**
1. Active nav link underline (`border-b-2` in emerald)
2. "Approved" section status badge (background + icon)
3. `/participate` nav link hover state
4. Emerald dot/icon on the "consultation summary available" indicator in portal section

**Never use accent for:** generic hover states on non-CTA elements, decorative dividers, card borders.

### Surface B — Workspace moderator review card (shadcn workspace)

Use existing shadcn semantic tokens — `bg-background`, `bg-muted`, `text-muted-foreground`, `text-foreground`, `border-border`. No new tokens.

Per-section status badge colors (using shadcn `variant` or inline):
- `pending`: muted background (`bg-muted`, `text-muted-foreground`) — slate/gray
- `approved`: emerald — `bg-emerald-50 text-emerald-700 border-emerald-200`
- `blocked`: destructive — `bg-red-50 text-red-700 border-red-200`
- `error`: warning orange — `bg-orange-50 text-orange-700 border-orange-200`

## Interaction Contracts

### Public Header

- **Active state:** current route link gets `border-b-2 border-[#179d53] text-[#000a1e]`. All other links: `text-[#44474e] hover:text-[#000a1e] transition-colors duration-300`.
- **Sticky behavior:** `sticky top-0 z-50` with `bg-[var(--cl-surface-container-low)]/80 backdrop-blur-md` (glassmorphism per `app/page.tsx` line 31 pattern).
- **Height:** 64px (`h-16`) — matches the existing layout stub.
- **Mobile breakpoint:** Nav links hidden below `md` (768px). Hamburger button shown. Menu expands as a full-width dropdown panel below the header (not slide-in overlay — simpler, zero animation library needed). Panel background: `var(--cl-surface)` with `border-b`.
- **Mobile menu animation:** CSS `max-height` transition from `0` to `auto` via `transition-[max-height] duration-200 ease-out` — no framer-motion, no additional JS.
- **Hamburger icon:** lucide `Menu` (open) / lucide `X` (close) — 24px, `text-[var(--cl-primary)]`.

### Moderator Review Card — Side-by-Side Layout

- **Layout:** Two-column grid at `md` breakpoint: `grid grid-cols-1 md:grid-cols-[1fr_320px] gap-6`. Left = prose editor. Right = source feedback panel.
- **Right panel (source feedback):** Fixed width 320px on desktop. Scrollable list of truncated feedback bodies (`line-clamp-3`). Shows: role badge (government / industry / etc.) + truncated body. Stakeholder names NOT shown (privacy enforcement per D-07 and Phase 9 PUB-05 precedent). Shows count: "4 accepted feedback items."
- **Left panel textarea:** Visible only when moderator clicks the edit icon on a section. Default shows rendered prose (non-editable). Click-to-edit reveals `<Textarea>` with the prose pre-filled. Save button replaces Approve for one interaction cycle. The edit icon button MUST carry `aria-label="Edit section summary"` — it is an icon-only control with no visible text label.
- **Per-section status badge:** Pill badge positioned in the section header row, right-aligned. Uses colors from Color section above.
- **Regenerate button:** `variant="outline"` Button, icon `lucide RefreshCw` 16px left of text. Sits in the section header row, left of the status badge.
- **Approve button:** `variant="default"` Button (workspace primary). Disabled when section is `approved`. After approval, button label becomes "Approved" with `lucide Check` icon and disabled state.
- **Loading skeleton:** While a section is `pending` with no prose (generation in-flight), replace the left panel content with `<Skeleton className="h-4 w-full" />` repeated 6 times at 8px gap. Right panel shows real feedback data (it loads independently).
- **Blocked section:** Left panel shows red-bordered callout (`border-red-200 bg-red-50 rounded-md p-4`) with blocked reason text. Regenerate button still active.

### Portal Section Summary Block

- **Position:** Immediately below the section's prose content, above the `<hr>` separator.
- **Container:** `rounded-lg border border-[var(--cl-outline-variant)] bg-[var(--cl-surface-container-low)] p-6 mt-8`
- **Approved state:** Shows `Stakeholder Perspectives` heading (16px semibold, Newsreader), subline, then the prose. Emerald accent dot (`w-2 h-2 rounded-full bg-[#179d53]`) inline before the heading.
- **Pending/blocked/error state:** Muted card. `bg-[var(--cl-surface-container)]` background. `Summary under review` heading in `text-[var(--cl-on-surface-variant)]`. No emerald accent.
- **No summary at all (version predates feature):** Render nothing — no placeholder.

### Framework Summary Block

- **Position:** Below `<WhatChangedLog>`, before the page closes.
- **Container:** Same as portal summary block above.
- **Heading:** `Consultation Summary` (20px semibold, Newsreader).
- **Content:** Approved summary prose sections, each labelled with the section title (14px semibold Inter). If a section has no approved summary, skip that section silently.
- **No published version:** Component not rendered at all per D-18.
