## Spacing Scale

Declared values (multiples of 4 — matches existing codebase patterns):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding, breadcrumb separator gap |
| sm | 8px | Compact element spacing, badge padding, tab trigger gap |
| md | 16px | Default element spacing, card inner padding, breadcrumb row padding-x |
| lg | 24px | Section padding, content area padding, tab bar bottom margin |
| xl | 32px | Layout gaps between major blocks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Breadcrumb row height: 36px (8px padding-y + 20px text line + 8px padding-y) — not a standard token, enforced as a fixed row.
- Tab bar height: 40px (`h-10`) — aligns with shadcn base-nova TabsList `h-8` plus 4px frame padding each side.
- "Give Feedback" button touch target: 36px minimum height (`size="sm"` in shadcn base-nova maps to h-8; use `size="default"` = h-9 for this CTA).
- Workspace nav link active underline bar: 2px — matches existing `after:h-0.5` in TabsTrigger line variant.

| Role | CSS Token | Light Value | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) — white | Page background, tab panel content area |
| Secondary (30%) | `--muted` / `--sidebar` | oklch(0.97 0 0) / oklch(0.985 0 0) | Section sidebar bg, breadcrumb row bg (use `--muted`), tab list bg (default variant) |
| Accent (10%) | `--primary` | oklch(0.205 0 0) — near-black | See reserved-for list below |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) — red | No destructive actions in Phase 13 — token carried over but unused |

**Accent (`--primary`) reserved exclusively for:**
1. Active tab trigger text + background highlight (`data-active:bg-background data-active:text-foreground`)
2. Active workspace nav link text (`font-semibold text-foreground` — existing pattern)
3. "Give Feedback" primary action button fill (variant="default" uses `--primary` bg)
4. Breadcrumb current-page segment (final non-linked crumb, rendered in `text-foreground` weight 500)
5. Focus ring on tab triggers and breadcrumb links (`--ring` token, which is `oklch(0.708 0 0)`)

**Muted foreground (`--muted-foreground`)** used for:
- Breadcrumb ancestor segments (non-current links)
- Breadcrumb separators (ChevronRight icon, 12px, `text-muted-foreground`)
- Inactive tab trigger labels
- Inactive workspace nav links (existing pattern)
- Tab count badges when zero
- Loading skeleton shimmer via `--muted`

## Breadcrumb Component Contract

### Structure
```
<nav aria-label="Breadcrumb">
  <ol class="flex items-center gap-1 text-sm">
    <li><Link href="..." class="text-muted-foreground hover:text-foreground">Ancestor</Link></li>
    <li><ChevronRight class="h-3 w-3 text-muted-foreground" /></li>
    <li><span class="font-medium text-foreground">Current Page</span></li>
  </ol>
</nav>
```

### Placement
Breadcrumb row sits directly below the `<header>` in workspace layout — between header border-b and `<main>`. Height: 36px. Background: `bg-muted/50`. Padding: `px-6 py-2`. Border-bottom: `border-b border-border`.

### Label Rules
- **Policies** → literal "Policies" (no entity load needed)
- **Policy title** → from data already loaded in layout/page (e.g., `document.title`). Show `<Skeleton className="h-4 w-32" />` while loading.
- **Section title** → from data already loaded. Show `<Skeleton className="h-4 w-24" />` while loading.
- **Sub-route labels** → literal: "Feedback", "Change Requests", "Traceability", "Versions"
- **Current page segment** → `font-medium text-foreground`, not a link
- **Ancestor segments** → `text-muted-foreground hover:text-foreground transition-colors`

### Separator
ChevronRight icon, `h-3 w-3`, `text-muted-foreground`. Not a bullet or slash — chevron only.

### Mobile
Breadcrumb collapses to show only the direct parent + current page on viewport < 640px (sm breakpoint). Middle ancestors hidden with `hidden sm:inline`. No ellipsis — simply truncate ancestors.

## Tab Bar Contract (Policy Sub-Pages)

### Tabs
Order: **Content** (default) | **Feedback** | **Change Requests** | **Versions** | **Traceability**

Content is always shown. Remaining tabs are role-gated per existing `canViewCR` / `canViewTrace` booleans.

### Variant
Use `TabsList` with `variant="line"` (the underline variant from `tabs.tsx`). This matches the document sub-page context — it is a navigation pattern within a page, not a panel switcher.

### Route Mapping
Each tab is a `<Link>` wrapping a `TabsTrigger`. Active state is determined by `usePathname()` matching, not by Tabs `value` prop. This makes it pure route-based navigation (decision D-06).

```
Content          → /policies/[id]               (exact match)
Feedback         → /policies/[id]/feedback      (startsWith)
Change Requests  → /policies/[id]/change-requests (startsWith)
Versions         → /policies/[id]/versions      (startsWith)
Traceability     → /policies/[id]/traceability  (startsWith)
```

### Placement
Tab bar renders inside `app/(workspace)/policies/[id]/layout.tsx`, below the policy title and breadcrumb — above the two-column content grid (sidebar + content area). Full width, `border-b border-border`, `px-6`, `pt-4 pb-0`.

### Section Sidebar Visibility
The `SectionSidebar` is only rendered when the current tab is "Content" (i.e., the `[id]/page.tsx` route). All other tabs render full-width content with no sidebar. This requires the sidebar to move from `page.tsx` into a conditional in `layout.tsx` or kept in `page.tsx` (Content tab) only. Decision D-08 locks this.

### Mobile
Below `lg` breakpoint, tab bar scrolls horizontally (`overflow-x-auto whitespace-nowrap`). No dropdown — horizontal scroll. Section sidebar converts to Select dropdown (existing pattern, stays in `page.tsx`).

## "Give Feedback" Button Contract

### Placement
Bottom of `SectionContentView`, after the section content body, before any existing editor toolbar. Renders only when:
1. User has `feedback:submit` permission (stakeholder or any role with section assignment)
2. A section is selected

### Appearance
```
<Button variant="default" size="default">
  <MessageSquare className="mr-2 h-4 w-4" />
  Give Feedback
</Button>
```

Full-width on mobile (`w-full sm:w-auto`). Positioned in a `mt-6 flex justify-end` container.

### Behavior
On click: opens existing feedback submission form (sheet panel from Phase 4) pre-filled with `sectionId`. No new modal — reuses `FeedbackSubmitSheet` with `defaultSectionId` prop.

## Interaction States

### Breadcrumb Link
- Default: `text-muted-foreground`
- Hover: `text-foreground` with `transition-colors duration-150`
- Focus-visible: `outline-none ring-2 ring-ring/50 rounded-sm`
- Current (final segment): `text-foreground font-medium` — not a link, no hover state

### Tab Trigger (line variant)
- Default: `text-foreground/60`
- Hover: `text-foreground`
- Active (current route): `text-foreground` + bottom 2px `--foreground` underline (after pseudo-element from tabs.tsx line variant)
- Focus-visible: `border-ring ring-[3px] ring-ring/50 outline-1 outline-ring`
- Disabled (role-gated, hidden): do not render — omit from DOM entirely (not disabled, hidden)

### "Give Feedback" Button
- Default: `bg-primary text-primary-foreground`
- Hover: `bg-primary/90`
- Focus-visible: `ring-[3px] ring-ring/50`
- Loading: replace icon with `Loader2 animate-spin`, label stays "Give Feedback"

### Workspace Nav Link (unchanged)
- Active: `font-semibold text-foreground`
- Inactive: `text-muted-foreground hover:text-foreground transition-colors`

## Accessibility

- Breadcrumb: `<nav aria-label="Breadcrumb">` wrapping `<ol>`. ChevronRight separators have `aria-hidden="true"`. Current page `<span>` has `aria-current="page"`.
- Tab bar: `role="tablist"` via base-ui TabsList. Each `TabsTrigger` is `role="tab"`. Active tab has `aria-selected="true"` (handled by base-ui).
- "Give Feedback" button: text label is sufficient — icon is decorative (`aria-hidden="true"` on MessageSquare).
- Global feedback tabs: same tab/tabpanel pattern via base-ui Tabs.
- All interactive elements meet 4.5:1 contrast ratio against their background (OKLCH near-black on white satisfies this).
