## Criterion 7: CR merge -> version publish -> Cardano anchor + consultation summary

**Status:** [ ] PASS / [ ] FAIL / [ ] GATED

### Walk Procedure

1. Admin navigates to the auto-drafted CR from Criterion 6
2. Review the CR content, approve it
3. Merge CR -> new `documentVersions` row created, CR status -> `merged`
4. Admin publishes the new version -> `version.published` event fires via `sendVersionPublished()`
5. Two Inngest functions fan out on `version.published`:

   **a) `consultation-summary-generate`:**
   - Fetches anonymized feedback per section
   - Calls Groq `llama-3.3-70b-versatile` for each section with feedback
   - Stores per-section summaries in `documentVersions.consultationSummary` JSONB
   - Pre-condition: `SELECT COUNT(*) FROM feedback_items WHERE status = 'accepted'` must be > 0

   **b) `version-anchor`:**
   - Computes `hashPolicyVersion()` content hash
   - Calls `checkExistingAnchorTx()` via Blockfrost (idempotency pre-check)
   - Submits CIP-10 label 674 Cardano transaction on preview-net
   - Confirm-loop: polls `isTxConfirmed()` up to 20 attempts, 30s each = up to 10 minutes
   - **Do NOT close Inngest dev server during confirmation window**

6. Wait for both Inngest runs to complete (version-anchor may take up to 10 minutes)
7. Verify `documentVersions` row has `tx_hash` and `anchored_at` populated
8. Verify Cardanoscan link resolves: `https://preview.cardanoscan.io/transaction/<txHash>`
9. Navigate to `/portal/<policyId>/consultation-summary`
10. Find sections with status `pending`, click **Approve** (moderator reviews LLM summary)

2. **Pre-condition:** Wait for Criterion 7 `version-anchor` to fully confirm before starting this step (Cardano wallet concurrency key `cardano-wallet` limits to 1 concurrent tx)
3. Admin navigates to the milestone in the workspace
4. Click **Mark Ready** -> `markReady` tRPC mutation fires `sendMilestoneReady()` event
5. `milestoneReadyFn` runs 5-step pipeline in Inngest Dev UI:
   - Step 1: `compute-hash` -- re-derives contentHash from all linked entities using `hashMilestone()` + child hash functions
   - Step 2: `persist-hash` -- sets status='anchoring', persists hash
   - Step 3: `check-existing-tx` -- Blockfrost metadata scan for prior tx with same hash (idempotency)
   - Step 4: `submit-tx` -- `buildAndSubmitAnchorTx()` submits 1.5 ADA self-referential tx with CIP-10 label 674 metadata
   - Step 5: `confirm-loop` -- polls `isTxConfirmed()` up to 20 times at 30s intervals
6. Wait for all 5 steps to complete (may take up to 10 minutes)
7. Milestone transitions: status='anchored', `txHash` and `anchoredAt` populated
8. Visit `/portal/<policyId>` -> verify **Verified State** badge appears with Cardanoscan link
9. Click the Cardanoscan link -> verify `https://preview.cardanoscan.io/transaction/<txHash>` resolves (may take 1-2 min after confirmation)
